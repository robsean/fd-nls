Befehl: country.sys

  COUNTRY.SYS ist die Hilfedatei, von der COUNTRY seine
  Informationen bezieht.
  COUNTRY.SYS muss in der CONFIG.SYS / FDCONFIG.SYS geladen werden.
  Alternativ kann COUNTRY.SYS waehrend des Betriebes mit DEVLOAD
  nachtraeglich geladen werden.

Syntax:

  - keine -

Optionen:

  - keine -

Kommentar:

  COUNTRY ermoeglicht/setzt die internationalen Features von DOS, indem
  es einen Laendercode auswaehlt und (nur mit dem instabilen / Entwick-
  lungskernel) optional auch eine Codepage und eine Landes-(COUNTRY-)
  Datei.
  Der stabile Kernel hingegen verwendet nur den nnn Wert. Fuer volle
  Unterstuetzung sollten Sie also den instabilen / Entwicklungskernel
  verwenden. Sie brauchen fuer letzteren auch eine COUNTRY.SYS Datei,
  waehrend der erstere das Dateinamenargument ignoriert und nur Datum/
  Uhrzeit/Zahlenformate setzt, indem er Werte verwendet, die fuer einige
  uebliche Laender in den Kernel hineinkompiliert wurden. Er setzt keine
  Sortierreihenfolge und Gross-/Kleinbuchstabenuebersetzungstabellen.
  NLSFUNC kann nur mit dem instabilen Kernel verwendet werden.
  COUNTRY.SYS hat fuer die folgenden Laender eine komplette NLS Info
  (COUNTRY, UCASE, LCASE, FUCASE, COLLATE, DBCS and YESNO):
