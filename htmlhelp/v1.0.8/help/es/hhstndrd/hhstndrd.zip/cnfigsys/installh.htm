Comando: installhigh

  INSTALLHIGH ejecuta un programa TSR (un programa que permanece en la
  memoria), sin un entorno, en lugar de usar algo como lh o loadhigh.
  INSTALLHIGH carga un programa TSR en memoria de alto nivel, mientras que
  INSTALL carga el programa TSR en memoria de bajo nivel o convencional.
  INSTALLHIGH es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  installhigh=[unidad][ruta]nombre_archivo [opciones]
              unidad          La letra de la unidad, por ejemplo C:
              ruta            El directorio, por ejemplo \ejemplo\
              nombre_archivo  El nombre de archivo del TSR para ejecutar,
                              por ejemplo, test.sys

Opciones:

  opciones  Estas son las opciones que se pasar n al TSR. No son opciones
            para el comando 'installhigh'.

Comentarios:

  INSTALLHIGH carga el programa especificado por la 
  unidad:\ruta\nombre_archivo. Generalmente se utiliza para cargar 
  programas TSR (terminales estaticos residentes) con un bloque de entorno
  m¡nimo. Las opciones son para el propio programa; consulte la 
  documentaci¢n incluida con su software en particular para conocer las
  opciones y el uso admitidos. Dado que INSTALLHIGH carga el controlador en
  una memoria de alto nivel, queda m s espacio libre en memoria baja o
  convencional para otros controladores o para programas. Para cargar un
  controlador o un programa en una memoria de alto nivel, primero tienes
  que cargar un controlador como (HIMEM), HIMEMX o JEMMEX en CONFIG.SYS / 
  FDCONFIG.SYS.
  INSTALLHIGH es un comando interno a kernel.sys y no necesita ning£n otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    installhigh=C:\fdos\bin\ctmouse.exe
