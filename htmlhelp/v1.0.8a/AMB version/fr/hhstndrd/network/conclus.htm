Mise en réseau de FreeDOS - Conclusion

  Il existe beaucoup d'applications réseau disponibles pour les
  utilisateurs de FreeDOS, certains gratuits, d'autres même en tant
  que Logiciel Libre. C'est une bonne nouvelle : de la diffusion de
  fichiers à la navigation sur le Web - tout est possible avec FreeDOS.
  Il y a cependant des inconvénients : une intégration transparente dans
  les réseaux d'autres systèmes d'exploitation
  est difficile à réaliser. Ni le SMB (réseaux Windows et GNU/Linux) ni
  le NFS (les réseaux  GNU/Linux, UNIX) ne fonctionnent sans défaut avec
  une machine DOS. Et les cartes d'interface réseau modernes ont presque
  arrêté de prendre en charge les pilotes de paquets qui étaient
  autrefois les pilotes par défaut pour les réseaux sous DOS. Les autres
  pilotes nécessitent des logiciels supplémentaires qui ne sont pas libres
  et qui ne peuvent être distribués librement. Donc il y a encore
  beaucoup de travail - pour les développeurs, mais aussi ceux qui
  voudraient aider cette documentation :
    * PPP n'est pas expliqué - quelqu'un possède-t-il encore un Modem ?
    * Un guide pour l'utilisation de FreeDOS avec le WiFi (wireless LAN)
  serait génial.
    * NFS n'est pas encore traité.
    * La mise en réseau de FreeDOS dans une machine virtuelle n'est
      pas expliquée ici.
    * Un examen plus approfondi du client MS aiderait beaucoup
      d'utilisateurs.
    * La façon d'utiliser Novell Netware avec FreeDOS n'est pas
      encore documentée.
  Comme c'est désormais un wiki, vous pouvez ajouter des informations et
  continuer ce travail. Amusez-vous bien.
