Mise en réseau de FreeDOS - Client MS

Où le trouver

    (Image du client MS 3.0)
  Les programmes de Microsoft "LAN manager" et "MS Client 3.0" peuvent
  être téléchargés depuis le site FTP de Microsoft et sont gratuits pour
  un "usage interne" mais leur redistribution n'est pas autorisée.
    * MS LAN manager Client (1998) peut être trouvé à :
        ftp://ftp.microsoft.com/bussys/Clients/LANMAN/ (*01)
    * MS Client 3.0 (1995) peut être trouvé à :
        ftp://ftp.microsoft.com/bussys/Clients/MSCLIENT/ (*02)

Client MS en tant que serveur

  Il est possible (même si ce n'est pas officiellement pris en charge)
  d'utiliser une mise à jour Microsoft pour ajouter des fonctionnalités
  serveur au client MS. Cette mise à jour est appelée WG1049.EXE et
  peut être téléchargée ici :
    ftp://ftp.microsoft.com/softlib/mslfiles/wg1049.exe (*16)
  En effet, cela ajoute des fonctionnalités comparables à celles de
  "l'extension groupe de travail pour DOS", qui n'est plus vendue par
  Microsoft.
    (Image : Module complémentaire de groupe de travail pour MS-DOS)

Critiques

  L'intégration d'une machine DOS au sein d'un groupe de travail SMB
  moderne avec Windows XP ou GNU/Linux est cependant limitée. Par exemple
  il ne semble pas être possible d'accéder à un partage DOS depuis
  GNU/Linux avec smbmount. Et l'utilisation du TCP/IP comme protocole
  dans MS Client consomme trop de mémoire et peut rendre le système DOS
  instable. Steven Baker remarque que (see: http://web.archive.org/web/
  20070314043508/http://www.ece.ubc.ca/~gillies/9802net.html (*17)),
  bien que le coeur du protocole TCP/IP est resté stable au fil des ans,
  les protocoles SMB de Microsoft ont changé selon les mises à jour et
  existent "dans une douzaine de dialectes différents". Donc la
  compatibilité est un problème. D'autres problèmes peuvent être causés
  par les méthodes d'authentification qui sont utilisées par Windows ou
  Samba. À part cela, le fait que Microsoft permette encore de
  télécharger et utiliser ce logiciel peut être considéré comme une
  bonne chose. Certains fichiers du paquet "Client MS" sont également
  essentiels à l'utilisation d'applications TCP/IP sur des pilotes NDIS
  courants.

Les alternatives

  * NFS
  * Vous pourriez utiliser un serveur FTP sous DOS qui fonctionnerait
    en arrière-plan (comme Datalight Sockets, ce qui est expliqué ici
    (cf. http://sourceforge.net/mailarchive/message.php?msg_name=
    48BC5752.3090303%40mainz-online.de (*18)) et un outil pour mapper
    un serveur FTP à une lettre de lecteur (pour Windows par exemple :
    netdrive - cf. http://www.netdrive.net/ (*19)) pour intégrer vos
    machines DOS à un réseau. GNU/Linux peut même monter les serveurs
    FTP sans nécessiter d'outil additionnel.
