Commande : lh (loadhigh)

  LH charge un programme dans l'espace mémoire haute.
  Comme LH charge le programme en mémoire haute, il reste
  davantage d'espace libre dans la mémoire basse ou conventionnelle
  pour les autres programmes.
  LH est une commande de FICHIER BATCH / AUTOEXEC.BAT.
  Elle peut également être utilisée sur la ligne de commande.

Syntaxe :

  LH [lecteur][chemin]nomfichier [paramètres]
  LH [/L:région1[,taillemin1][;région2[,taillemin2]...] [/S] [/?]]
     [lecteur][chemin]nomfichier [paramètres]
     lecteur     La lettre du lecteur, par exemple C:
     chemin      Le répertoire, par exemple \exemple\
     nomfichier  Le nom du programme à charger dans l'espace mémoire
                 haute, par exemple edit.exe

Options :

  /L:région1[,taillemin1][;région2[,taillemin2]]...
                Spécifie la ou les régions de mémoire dans lesquelles
                charger le programme. Région1 spécifie le numéro de
                la première région de mémoire ; taillemin1 spécifie la
                taille minimale, si elle existe, pour la région1.
                Région2 et taillemin2 spécifient le numéro et la taille
                minimale de la seconde région, si elle existe. Vous
                pouvez spécifier autant de régions que vous le
                souhaitez.
  /S            Réduit un UMB à sa taille minimale lors du chargement
                du programme.
  /?            Affiche l'aide
  paramètres    Ce sont les options du programme que vous chargez.

Commentaires :

  LH peut également être utilisée en dehors d'un fichier batch.
  Pour charger un pilote ou un programme dans la mémoire haute, vous
  devez d'abord charger un pilote comme (HIMEM), HIMEMX, JEMMEX, (EMM386)
  ou JEMM386 dans config.sys. Vous pouvez aussi utiliser LOADHIGH au
  lieu de LH.
  LH est une commande interne à command.com et ne nécessite aucun
  autre fichier pour fonctionner.

Exemples :

  DANS UN FICHIER .BAT / AUTOEXEC.BAT :
    lh c:\fdos\bin\edit.exe
