Commande : jemmex

  JEMMEX est un gestionnaire de mémoire étendue qui comprend déjà un
  XMM. JEMMEX doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  DEVICE=[lecteur][chemin][jemmex.exe] [options]
         lecteur  La lettre du lecteur, par exemple C:
         chemin   Le répertoire, par exemple \exemple\
  ou lancez-le depuis la ligne de commande :
  [lecteur][chemin][jemmex.exe] [options]

Options :

 +A20/NOA20     Active ou désactive l'émulation de la désactivation de
                A20 (valeur par défaut : activée).
  A20METHOD:m   Défini la méthode de basculement A20. Les valeurs
                possibles pour m sont KBC, PS2, BIOS, FAST, PORT92
                and ALWAYSON.
  ALTBOOT       Utilise une stratégie alternative de redémarrage.
  B=####        Spécifie l'adresse de segment le plus bas pour la
                banque EMS (par défaut=4000).
  D=###         Règle la taille du tampon DMA en ko (par défaut=64,
                max 128).
  EMX           Compatibilité accrue avec l'extenseur DOS EMX.
  FASTBOOT      Redémarrage rapide. Nécessite que Jemm386 soit chargé
                dans CONFIG.SYS.
  FRAME=E000    Définit le cadre de page EMS (FRAME=NONE désactive le
                cadre). Toutes les valeurs entre 8000 et E000 sont
                acceptées, mais toutes ne fonctionneront pas.
  HMAMIN=n      Définit la quantité minimale en Ko pour allouer le HMA.
  I=D000-D7FF   Force l'utilisation d'une région pour les UMB. Sans cette
                option la plage C000-EFFF est analysée pour trouver des
                pages inutilisées. Peut aussi être utilisée pour ajouter
                (des parties de) A000-BFFF ou F000-F7FF en tant qu'UMB.
                N'utilisez pas cette option si vous ne savez pas ce que
                vous faites !
  I=TEST        Recherche des pages inutilisées dans les ROM, inclut les
                régions trouvées en tant qu'UMB.
  [MAX=]#####   Limite pour la mémoire VCPI (et EMS si < 32M) en ko
                (la valeur par défaut est de 120 Mo).
  MAXEXT=n      Limite l'utilisation de la mémoire étendue à n ko.
  MIN=#####     Réserve jusqu'à ##### ko pour la mémoire EMS/VCPI sur
                init (par défaut=0).
  NOCHECK       Interdit l'accès à l'espace d'adressage sans RAM (MMIO).
  NOE801        N'utilise pas Int 15h, E801h pour obtenir la quantité de
                mémoire étendue.
  NOE820        N'utilise pas Int 15h, E820h pour otenir la quantité de
                mémoire étendue.
  NOEMS         Désactive la gestion EMS.
  NODYN         Pas d'allocation dynamique de mémoire XMS (utilisez
                MIN= pour définir la quantité fixe).
  NOHI          Ne déplace pas la partie résidente dans le premier UMB.
  NOINVLPG      N'utilise pas l'opcode INVLPG.
 +PGE/NOPGE     Active ou désactive l'utilisation de la fonctionnalité
                Page Global Enable (par défaut : désactivée).
  S=D000-D7FF   Suppose que la Shadow-RAM est activée par UMBPCI,
                l'inclus en tant qu'UMB.
  SB            Mode de compatibilité avec le pilote SoundBlaster.
  SPLIT         Récupère les pages EPROM 4ko partiellement utilisées
                pour les UMB.
 +VCPI/NOVCPI   Support de VCPI activé/désactivé (par défaut : activé).
  VDS/NOVDS     Services de Virtual DMA activés/désactivés (par défaut :
                activés).
 +VME/NOVME     Extentions Mode V86 activées/désactivées (par défaut :
                activées).
  VERBOSE       Affiche des détails supplémentaires pendant le démarrage
                (forme abrégée : /V).
  X=D000-D7FF   Interdit à Jemm386 de toucher ou d'utiliser la région.
  X=TEST        Analyse de la plage de mémoire C000-EFFF pour exclure
                les UMB.
  X2MAX=n       Valeur max en Ko pour le rapport de mémoire libre XMS V2
                (par défaut=65535).
  XMSHANDLES=n  Nombre d'indicateurs XMS disponibles (8<=n<=128,
                défaut=32).
  /?            Affiche l'aide.
 '+': cette option peut être définie de manière dynamique en lançant
      Jemm386 depuis la ligne de commande.
  Lorsqu'il est appelé depuis la ligne de commande, Jemmex comprendra
  en outre:
    LOAD        installe
    UNLOAD      désinstalle

Commentaires :

  JEMMEX nécessite un processeur 80386+. Chargez JEMMEX comme
  DEVICE=JEMMEX.EXE dans CONFIG.SYS (bien que ce soit un fichier
  ".exe" !) JEMMEX doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.
  JEMMEX est probablement un meilleur choix que HIMEMX / JEMM386 car
  il aura besoin de moins de mémoire q'un XMM externe + JEMM386.

Exemples:

  DANS CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\jemmex.exe
