Commande : append

  APPEND permet aux programmes d'ouvrir des fichiers de données dans les
  dossiers spécifiés comme s'ils étaient dans le répertoire courant.

Syntaxe :

  APPEND [[lecteur]chemin[;...]] [/X[:ON|:OFF]] [/chemin:ON|/chemin:OFF]
         [/E]
  APPEND ;
  APPEND [/?]
         lecteur  Lettre de lecteur, par ex. C:
         chemin   Répertoire à ajouter, par ex. \exemple

Options :

  /X[:ON]    Étendre APPEND aux recherches et exécutions de commandes.
  /X:OFF     Applique APPEND seulement aux requêtes d'ouverture de
             fichiers. /X:OFF est la valeur par défaut.
  /PATH:ON   Rechercher dans les dossiers ajoutés pour les requêtes
             de fichiers qui comprennent déjà un chemin. C'est la
             valeur par défaut.
  /PATH:OFF  Désactive /PATH:ON.
  /E         Stocke le dossier ajouté dans l'environnement.
             /E ne peut être utilisé que dans la première invocation
             d'APPEND. Vous ne pouvez pas inclure des chemins sur
             la même commande que /E.
  APPEND ;   nettoie la liste des répertoires ajoutés.
  APPEND     sans paramètres affiche la liste des répertoires ajoutés.
  /?         Affiche l'aide.

Commentaires :

  N'utilisez pas APPEND avec Windows ou les extenseurs 32 bits.
  APPEND s'installe lui-même comme une commande interne après sa première
  exécution. Les invocations suivantes DOIVENT exclure le chemin de
  fichier et l'extension de la commande APPEND.
  APPEND prend en charge le NLS (support des langues nationales).

Exemples :

    Première utilisation :
  C:\command\append.exe D:\Data\df1.dat
    Utilisation parfaitement légitime de la commande Append pour la
    première invocation.
    Seconde utilisation erronée :
  C:\command\append.exe D:\Data\df2.dat
    Seconde utilisation invalide. Le résultat est que le nouvel append
    chargé par command.com se voit dans la mémoire et quitte avec une
    erreur. 
    Seconde utilisation valide :
  append D:\Data\df2.dat
    Seconde utilisation valide. Command.com regarde sa liste de commandes
    internes, voit append et l'exécute en lui transmettant le restant de
    la ligne de commande.
