Command: himem

  HIMEM est le gestionnaire de mémoire haute.
  HIMEM doit être charge dans CONFIG.SYS / FDCONFIG.SYS.
  HIMEM est périmé. Vous devriez utiliser HIMEMX ou JEMMEX à la place.

Syntaxe :

  HIMEM [/MAX=####] [/METHOD:xxx] [/HMAMIN=n] [/NUMHANDLES=m]
        [/TESTMEM:ON|OFF] [/VERBOSE] [/NOABOVE16] [/X] [/LOG]

Options :

  /MAX=#####       Limite la mémoire contrôlée par HIMEM à ##### Ko.
  /METHOD:xxx      Spécifie la méthode à utiliser pour la gestion A20.
                   Valeurs possibles pour xxx :
                   ALWAYSON    Suppose que la ligne A20 est toujours
                               allumée
                   BIOS        Utilise le BIOS pour basculer la ligne A20
                   FAST        Utilise le port 92h, contourne le test
                               INT 15h
                   PS2         Utilise le port 92h, contourne le test
                               PS/2
                   KBC         Utilise le contrôleur de clavier
                   PORT92      Utilise toujours le port 92h
  /HMAMIN=n        Spécifie le nombre minimal de Ko de HMA qu'un
                   programme doit demander pour obtenir l'accès au
                   HMA (par défaut : 0Ko).
  /NOX2MAX32       Ne limite pas le rapport de mémoire XMS 2.0
                   libre/dispo. à 32Mo-1Ko
  /NUMHANDLES=m    Spécifie le nombre d'indicateurs XMS disponibles
                   (déf: 72).
  /TESTMEM:ON|OFF  Effectue ou ignore le test de mémoire étendue 
                   (déf: OFF).
  /VERBOSE         Donne des informations supplémentaires pendant
                   l'installation.
  /NOABOVE16       N'utilise pas la fonction E801 d'INT 15h pour
                   détecter > 64Mo.
  /X               N'utilise pas la fonction E820h d'INT 15h pour
                   détecter > 64M.
  /X2MAX32         Limite le rapport de mémoire XMS 2.0 libre/disponible
                   à 32Mo-1Ko (par défaut).
  /LOG             Enregistre l'activité du pilote dans un fichier
                   journal.
  /?               Affiche l'aide.

Commentaires :

  HIMEM nécessite un processeur 80386+. EMM386 peut être chargé après
  ce pilote. Chargez HIMEM comme DEVICE=HIMEM.EXE dans CONFIG.SYS
  (bien que ce soit un fichier ".exe" !)
  HIMEM doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himem.exe
    device=c:\fdos\bin\emm386.exe
