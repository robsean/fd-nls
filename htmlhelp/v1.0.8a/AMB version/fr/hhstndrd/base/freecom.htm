Commande : FreeCom (command.com)

  FREECOM (command.com) démarre une nouvelle copie du shell de commande
  de FreeDOS.

Syntaxe :

  COMMAND [[lecteur]chemin] [device] [/E:nnnnn] [/L:nnnn] [/U:nnn] [/P]
          [/MSG] [/LOW] [/Y [/[C|K] commande]]
  COMMAND [/?]
          [lecteur]chemin  Spécifie le répertoire contenant COMMAND.COM,
                           par exemple C:\exemple\
          lecteur          La lettre du lecteur, par exemple C:
          chemin           Le répertoire, par exemple \exemple\
          device           Spécifie le périphérique à utiliser pour les
                           entrées et sorties de commandes.

Options :

  /E:nnnnn    Règle la taille initiale de l'environnement à nnnnn octets.
              (nnnnn devrait être entre 256 et 32,768).
  /L:nnnn     Spécifie la taille des tampons internes (nécessite /P
              également). (nnnn devrait être entre 128 et 1,024).
  /U:nnn      Spécifie la taille du tampon d'entrée (nécessite /P
              également). (nnn devrait être entre 128 et 255).
  /P          Rend le nouveau shell de commandes permanent (impossible
              de sortir).
  /MSG        Stocke tous les messages d'erreur dans la mémoire
              (nécessite /P également).
  /LOW        Force le shell de commandes à garder ses données résidentes
              dans mémoire basse.
  /Y          Parcourt le programme batch spécifié par /C o.
  /C commande Exécute la commande spécifiée puis retourne.
  /K commande Exécute la commande spécifiée puis continue.
  /?          Affiche l'aide.

Commentaires :

  [lecteur]chemin  Le lecteur et le chemin où le shell doit rechercher la
                   partie transitoire du programme. C'est généralement
                   seulement nécessaire pour définir le COMSPEC.
  device           Un périphérique alternatif pour toutes les E/S.
  /e:nnnnn         La taille de l'environnement, en octets, dans la plage
                   160-32768. Ce nombre sera arrondi aux 16 octets les
                   plus proches. La valeur par défaut est 256.
  /p               Rend le shell permanent, donc la commande EXIT ne
                   quitte pas le shell.
  /c {chaîne}      Exécute la commande dans {chaîne}, puis quitte.
  /msg             Charge dans la mémoire les messages d'erreur qui
                   pourraient être stockés sur le disque.
  Command (command.com) est aussi appelé FreeCom. Il contient de
  nombreuses commandes internes qui n'ont besoin d'aucun autre fichier
  pour fonctionner.
  Vous pouvez trouver la liste des commandes internes ici.

Exemples :

  C:\command /E:1024 /L:1024 /U:128 /P
  command /?
