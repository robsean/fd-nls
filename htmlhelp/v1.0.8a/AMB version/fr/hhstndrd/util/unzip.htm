Commande : unzip

  UNZIP liste, teste ou extrait des fichiers depuis une archive ZIP,
  trouvée couramment sur les systèmes DOS. Le comportement par défaut
  (sans option) est d'extraire tous les fichiers depuis l'archive ZIP
  spécifiée dans le répertoire courant (et les sous-répertoires en
  dessous).

Syntaxe :

  unzip [-Z] [-options[modificateurs]] fichier[.zip] [fichiers]
        [-x xlist] [-d exdir]
          fichier[.zip]  Chemin des archives ZIP. Cela peut être un
                         joker.
          [fichiers]   Une liste facultative des membres d'archive qui
                       doivent être traités, séparés par des espaces.
          [-d exdir]   Un répertoire facultatif où les fichiers seront
                       extraits. Par défaut, tous les fichiers et les
                       sous-répertoires sont recréés dans le répertoire
                       courant ; l'option -d permet l'extraction dans
                       un répertoire arbitraire.

Options :

    -d   Extraire les fichiers dans exdir (un répertoire optionnel).
    -f   Rafraîchir les fichiers existants, c'est-à-dire extraire
         seulement les fichiers qui existent déjà sur le disque et
         qui sont plus récents que les copies du disque.
    -l   Lister les fichiers (format court). Les noms, tailles des
         fichiers décompressées et l'horodatage de modification des
         fichiers spécifiés, sont affichés avec les totaux pour tous
         les fichiers spécifiés.
    -p   Extraire les fichiers vers le pipe (sortie standard), aucun
         message.
    -t   Tester les données de l'archive compressée. Cette option
         extrait chaque fichier spécifié dans la mémoire et compare
         la somme de contrôle CRC du fichier décompressé avec la valeur
         CRC stockée du fichier original.
    -u   Mettre à jour les fichiers, en créer de nouveaux si nécessaire.
    -x   Exclure les fichiers suivants (dans xlist). Xlist est une liste
         optionnelle de membres d'archive à exclure du traitement. Comme
         les caractères joker correspondent normalement aux ('/')
         séparateurs de répertoire, cette option peut être utilisée
         pour exclure tout fichier qui serait dans un sous-répertoire.
    -z   Afficher le commentaire de l'archive.
    -Z   Mode ZipInfo. Si la première option sur la ligne de commande
         est -Z, les options restantes sont considérées comme des
         options zipinfo(1L).
  Modificateurs :
    -$   Étiqueter les amovibles (-$$ => disques fixes)
    -a   Convertir automatiquement tout fichier texte.
    -aa  Traiter TOUS les fichiers comme des fichiers textuels.
    -C   -C Correspondance insensible à la casse pour les noms de
         fichiers.
    -j   Chemins indésirables. La structure de répertoires de
         l'archive n'est pas recréée ; tous les fichiers sont déposés
         dans le répertoire d'extraction.
    -L   Basculer (certains) noms en minuscules.
    -n   Ne jamais écraser les fichiers existants.
    -o   Écraser les fichiers SANS demande de confirmation.
    -M   Envoyer vers le pipe de pagination "more".
    -q   Mode silencieux (-qq => encore plus silencieux).
    -s   Convertir les espaces dans les noms de fichiers en '_'
    -v   Être verbeux ou afficher les informations de diagnostic
         de version.
    -V   Conserver les numéros de version VMS.

Commentaires :

  Pour davantage d'informations veuillez lire UNZIP.TXT.

Exemples :

  unzip data1 -x joe    extraire tous les fichiers sauf joe depuis
                        le fichier zip data1.zip
  unzip -fo foo ReadMe  remplacer silencieusement le ReadMe existant
                        si le fichier de l'archive est plus récent
