Commande : devicehigh

  DEVICEHIGH charge un pilote dans la mémoire haute, tandis que DEVICE
  charge le pilote dans la mémoire basse ou conventionnelle.
  DEVICEHIGH est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  device=[lecteur][chemin]nomfichier [options]
         lecteur    La lettre du lecteur, par exemple C:
         chemin     Le répertoire, par exemple \exemple\
         nomfichier Le (fichier de) pilote à charger, par exemple
                    himemx.exe

Options :

  options  Il s'agit là d'options pour le programme du
           pilote, pas pour la commande 'device'.

Commentaires :

  DEVICEHIGH charge le pilote de périphérique spécifié par
  lecteur:\chemin\nomfichier dans la mémoire haute [AU-DESSUS DE 640 Ko]
  (si cela ne marche pas, il devrait le charger dans la mémoire
  conventionnelle).
  Les options sont pour le pilote, veuillez vous référer à la
  documentation de votre périphérique pour les options prises en
  charge et leur utilisation.
  Comme DEVICEHIGH charge le pilote dans la mémoire haute il reste
  davantage d'espace libre dans la mémoire basse ou conventionnelle
  pour les autres pilotes ou programmes. Pour charger un pilote ou
  un programme dans la mémoire haute vous devez d'abord charger un
  pilote comme (HIMEM), HIMEMX ou JEMMEX dans CONFIG.SYS / FDCONFIG.SYS.
  DEVICEHIGH est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    DEVICE=C:\FDOS\BIN\XMGR.SYS /N128 /B
    DEVICEHIGH=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
  DANS UN MENU :
    23?DEVICEHIGH=C:\FDOS\BIN\XMGR.SYS /N128 /B
    23?DEVICEHIGH=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
