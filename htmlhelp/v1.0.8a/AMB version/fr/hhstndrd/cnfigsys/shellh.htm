Commande : shellhigh

  SHELLHIGH indique à FreeDOS quel shell de commande il doit exécuter.
  SHELLHIGH est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntax:

  shellhigh=[lecteur][chemin]nomfichier [options]
            lecteur     La lettre du lecteur, par exemple C:
            chemin      Le répertoire, par exemple \exemple\
            nomfichier  Le shell de commande à exécuter. Par exemple,
                        C:\exemple\COMMAND.COM

Options :

  options  Celles-ci sont transmises au shell de commande.

Commentaires :

  SHELLHIGH indique le shell à utiliser ; souvent pour modifier le
  comportement de COMMAND.COM. Note : c'est command.com qui
  traite AUTOEXEC.BAT ; avec l'option SHELL vous pouvez faire
  en sorte que command.com traite un fichier nommé file différemment
  (tel que FDAUTO.BAT afin de coexister avec un autre DOS qui
  utiliserait d'autres options de configuration) ou pour exécuter
  un interpréteur de commandes entièrement différent tel que 4DOS ou
  une variante de sh (UNIX).
  SHELLHIGH ressemble à SHELL, sauf qu'il tente de charge le SHELL de
  commande (COMMAND.COM) d'abord dans la mémoire haute (en cas d'échec
  il devrait le charger dans la mémoire conventionnelle).
  Remarque : L'ordre avec lequel vous chargez les périphériques peut
  avoir un grand impact sur la quantité de mémoire libre disponible.
  En général essayez de charger les gros (en termes d'utilisation de
  mémoire) programmes d'abord dans la mémoire haute.
  Important : vous devriez avoir un gestionnaire de mémoire tel que
  (HIMEM), HIMEMX ou JEMMEX d'installé (device=himem.exe ou
  device=himemx.exe avant dans CONFIG.SYS / FDCONFIG.SYS) avant
  d'utiliser cette option.
  SHELLHIGH est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
   shellhigh=C:\4DOS.COM /E:256 /P
   shellhigh=C:\FDOS\bin\command.com C:\FDOS\bin /E:1024 /P=C:\fdauto.bat
