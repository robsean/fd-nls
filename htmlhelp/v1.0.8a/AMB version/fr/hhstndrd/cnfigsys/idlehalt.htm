Commande : idlehalt

  IDLEHALT active la fonctionnalité d'économie d'énergie intégrée au
  kernel si n n'est pas égal à 0.
  IDLEHALT est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  idlehalt=n
    où n peut être égal à -1, 0, 1 ou une valeur plus élevée (par
    défaut : 0)

Options :

  - aucune -

Commentaires :

  IDLEHALT active la fonctionnalité d'économie d'énergie intégrée au
  noyau si n n'est pas égal à 0. La valeur -1 active tous les hooks,
  1 active seulement les hooks "sûrs", le microprocesseur est arrêté
  seulement si le kernel attend l'entrée du périphérique CON char.
  D'autres hooks pour n = -1 et n > 0 dépendent de la version du
  noyau : en plus des hooks sûrs, d'autres hooks peuvent être activés,
  par exemple un pour int 0x2f, ax=0x1680
   "release time slice".
  Remarque : dans de rares cas, l'entrée ou la sortie du mode HLT (qui
  provoque de grands changements de consommation électrique du
  microprocesseur) peuvent provoquer des plantages si les blocs
  d'alimentation ou les cartes mères bon marché ne peuvent pas filtrer
  correctement les états transitoires. Baisser la fréquence d'horloge
  de tout le système peut aider dans ce cas.
  Linux effectue toujours quelques HLT au moment de l'amorçage, pour
  forcer un plantage précoce des systèmes bogués (démarrez avec no-hlt
  pour désactiver HLT).
  P90 peut avoir un HLT bogué ? www.tavi.co.uk/ps2pages/ohland/halt.html
  IDLEHALT est une commande interne à kernel.sys et ne nécessite aucun
  autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    idlehalt=1
