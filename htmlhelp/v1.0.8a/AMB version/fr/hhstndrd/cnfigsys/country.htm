Commande : country

  COUNTRY définit le comportement international de DOS.
  COUNTRY est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  country=nnn[,[mmm]][,[lecteur][chemin]nomfichier]
          lecteur    La lettre du lecteur, par exemple C:
          chemin     Le répertoire, par exemple \exemple\
          nomfichier Le nom du fichier d'informations sur le pays.
                     S'il n'est pas spécifié, le fichier 'COUNTRY.SYS'
                     est utilisé à sa place.

Options :

  nnn    Le code pays, comme indiqué sur le tableau ci-dessous (001=ÉU).
  mmm    La page de code, comme indiqué sur le tableau ci-dessous (437
         est la valeur par défaut, 850 est la forme mise à jour, 1252
         pour Windows).
         Si elle n'est pas spécifiée, une page de code sera choisie
         automatiquement.

  Codes pays

    * HH signifie une heure de 24 heures
    * hh une heure de 12 heures.
    * X signifie que AM et PM sont ajoutés

Commentaires :

  COUNTRY active/définit les fonctionnalités internationales de DOS en
  choisissant un code pays et (seulement avec un kernel instable / de
  développement) optionnellement une page de code et un fichier de pays.
  Le kernel stable utilise seulement la valeur nnn. Pour une prise en
  charge complète, utilisez le kernel instable / de développement. Vous
  aurez aussi besoin d'un fichier COUNTRY.SYS pour ce dernier, alors que
  le premier ignore l'argument de nom de fichier et ne définit que
  le format de date/heure/numérotation en utilisant les valeurs compilées
  dans le kernel pour quelques pays. Il ne définit pas l'ordre de tri et
  les tableaux de traduction minuscules / majuscules. NLSFUNC ne peut
  qu'être utilisé avec le kernel instable.
  COUNTRY est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himemx.exe
    device=c:\fdos\bin\jemm386.exe
    dos=high,umb
    country=001,437,c:\fdos\bin\country.sys
