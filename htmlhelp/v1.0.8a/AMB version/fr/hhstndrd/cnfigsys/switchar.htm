Commande : switchar

  SWITCHAR définit un caractère de commutateur.
  SWITCHAR est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  switchar=c
    Définit le switchar par défaut sur le caractère c.

Options:

  - aucune -

Commentaires :

  SWITCHAR définit le caractère de commutateur par défaut sur le
  caractère c donné, par exemple "-". Où c est un seul caractère
  utilisé pour indiquer un paramètre de ligne de commande dans un
  commutateur d'option. La valeur par défaut est la barre oblique (/).
  Remarque : Cela définit simplement la valeur renvoyée par une
  requête get switchar, cela n'affectera pas les programmes qui
  utilisent des caractères de commutation codés en dur.
  Le caractère de commutation est le signe avant une option pour
  que DOS sache que le texte suivant est une option, par exemple
  le signe "/" dans dir /w.
  Les caractères de commutation typiques sont par exemple "-", "/".
  SWITCHAR est utilisé par command.com et certains mais pas tous les
  pilotes et commandes externes.
  SWITCHAR est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS :
    switchar=-
  Au lieu de
    dir /p vous auriez alors à entrer
    dir -p .
