Commande : shell

  SHELL indique à FreeDOS quel shell de commande il doit exécuter.
  SHELL est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  shell=[lecteur][chemin]nomfichier [options]
        lecteur     La lettre du lecteur, par exemple C:
        chemin      Le répertoire, par exemple \exemple\
        nomfichier  Le shell de commande à exécuter. Par exemple,
                    C:\exemple\COMMAND.COM

Options :

  options  Celles-ci sont transmises au shell de commande.

Commentaires :

  SHELL indique le shell à utiliser ; souvent pour modifier le
  comportement de COMMAND.COM. Note : c'est command.com qui
  traite AUTOEXEC.BAT ; avec l'option SHELL vous pouvez faire
  en sorte que command.com traite un fichier nommé file différemment
  (tel que FDAUTO.BAT afin de coexister avec un autre DOS qui
  utiliserait d'autres options de configuration) ou pour exécuter
  un interpréteur de commandes entièrement différent tel que 4DOS ou
  une variante de sh (UNIX).
  SHELL est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS :
    shell=C:\4DOS.COM /E:256 /P
    shell=C:\FDOS\bin\command.com C:\FDOS\bin /E:1024 /P=C:\fdauto.bat
