Commande : device

  DEVICE charge un pilote dans la mémoire basse ou conventionnelle,
  tandis que
  DEVICEHIGH charge le pilote dans la mémoire haute.
  DEVICE est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  device=[lecteur][chemin]nomfichier [options]
         lecteur    La lettre du lecteur, par exemple C:
         chemin     Le répertoire, par exemple \exemple\
         nomfichier Le (fichier de) pilote à charger, par exemple
                    himemx.exe

Options :

  options  Il s'agit là d'options pour le programme du
           pilote, pas pour la commande 'device'.

Commentaires :

  DEVICE charge le pilote de périphérique spécifié par
  lecteur:\chemin\nomfichier dans la mémoire conventionnelle [640 Ko].
  Les options sont pour le pilote ; veuillez vous référer à la
  documentation de votre périphérique pour les options prises en charge
  et leur utilisation.
  Comme DEVICE charge le pilote dans la mémoire basse ou conventionnelle,
  cela laisse moins d'espace mémoire pour les autres pilotes ou
  programmes dans la mémoire basse ou conventionnelle. Cela peut
  provoquer des problèmes lors de l'exécution des programmes ou le
  chargement d'autres pilotes.
  DEVICE est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    DEVICE=C:\FDOS\BIN\XMGR.SYS /N128 /B
    DEVICE=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
  DANS UN MENU :
    23?DEVICE=C:\FDOS\BIN\XMGR.SYS /N128 /B
    23?DEVICE=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
