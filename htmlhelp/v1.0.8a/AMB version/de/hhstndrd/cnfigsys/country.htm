Befehl: country

  COUNTRY stellt das internationale Verhalten von DOS ein.
  COUNTRY ist ein CONFIG.SYS / FDCONFIG.SYS-Befehl.

Syntax:

  country=nnn[,[mmm]][,[[Laufwerk][Pfad][Dateiname]]]
          Laufwerk   Der Laufwerksbuchstabe, z.B: C:
          Pfad       Der zu wählende Pfad, z.B.: \Beispiel\
          Dateiname  Der Dateiname der Länderschlüsseldatei.
                     Wenn keiner angegeben wird, wird stattdessen
                     "COUNTRY.SYS" verwendet.

Optionen:

  nnn       Der Länderschlüssel wie in der Tabelle unten beschrieben
            (in der Regel die internationale Telefonvorwahl; 001=US).
  mmm       Die Codepage, wie in der Tabelle unten beschrieben. (437 ist 
            Standard, 850 ist upgedatete Form, 1252 für Windows).
            Wenn keine angegeben wird, wird automatisch eine Codepage
            gewählt.

  Länderschlüssel:
  In der Tabelle bedeuten:

    * HH eine 24-Stunden-Stunde
    * hh eine 12-Stunden-Stunde
    * X  AM und PM werden angehängt (AM=morgens, PM=abends)

Kommentar:

  COUNTRY ermöglicht/setzt die internationalen Features von DOS, indem
  es einen Ländercode auswählt und (nur mit dem instabilen / Entwick-
  lungskernel) optional auch eine Codepage und eine Landes-(COUNTRY-)
  Datei.
  Der stabile Kernel hingegen verwendet nur den nnn Wert. Für volle
  Unterstützung sollten Sie also den instabilen / Entwicklungskernel
  verwenden. Sie brauchen für letzteren auch eine COUNTRY.SYS Datei, 
  während der erstere das Dateinamenargument ignoriert und nur Datum/
  Uhrzeit/Zahlenformate setzt, indem er Werte verwendet, die für einige
  übliche Länder in den Kernel hineinkompiliert wurden. Er setzt keine
  Sortierreihenfolge und Gross-/Kleinbuchstabenübersetzungstabellen.
  NLSFUNC kann nur mit dem instabilen Kernel verwendet werden.
  COUNTRY ist ein interner Befehl, d.h. der Befehl ist in der kernel.sys
  enthalten. Es wird zur Ausführung des Befehls kein zusätzlicher
  Befehl (Datei) benötigt.

Beispiel:

  IN DER CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himemx.exe
    device=c:\fdos\bin\jemm386.exe
    dos=high,umb
    country=001,437,c:\fdos\bin\country.sys
