Befehl: himemx

  HIMEMX ist ein Verwalter für den hohen Speicherbereich.
  HIMEMX muß in der CONFIG.SYS / FDCONFIG.SYS geladen werden.

Syntax:

  HIMEMX [/MAX=####] [/METHOD:xxx] [/HMAMIN=n] [/NUMHANDLES=m]
        [/TESTMEM:ON|OFF] [/VERBOSE] [/NOABOVE16] [/X] [/LOG]

Optionen:

  /MAX=#####       Begrenzt den von HIMEM kontrollierten Speicher 
                   auf #####KB.
  /METHOD:xxx      Spezifiziert die Methode, die für das A20 "Handling"
                   verwendet wird.
                   Mögliche Werte für xxx:
                   ALWAYSON    Nimmt an, daß die A20 Leitung 
                               dauernd AN ist.
                   BIOS        Verwendet das BIOS um die A20 
                               Leitung zu steuern.
                   FAST        Verwendet Port 92h, umgeht 
                               den INT 15h Test.
                   PS2         Verwendet Port 92h, umgeht
                               den PS/2 Test.
                   KBC         Verwendet den Tastaturcontroller.
                   PORT92      Verwendet IMMER Port 92h.
  /HMAMIN=n        Spezifiziert die Mindestgröße an HMA in KB, die
                   ein Programm anfordern muß, um Zugang zur HMA zu
                   bekommen (Standard: 0 KB).
  /NOX2MAX32       Keine Begrenzung des Speicherbereichs für XMS 2
                   (Standard).
  /NUMHANDLES=m    Setzt die maximale Anzahl der "Handles", die für
                   Programme erhältlich sind. (Standard ist 48).
  /TESTMEM:ON|OFF  Führt einen umfangreichen Speichertest aus (ON) oder
                   übergeht ihn (OFF). Standard ist OFF.
  /VERBOSE         Gibt während der Installation Zusatzinformationen
                   aus. Hilfreich für Fehlerberichte.
  /NOABOVE16       Verwendet nicht die INT 15h Funktion E801h um Speicher
                   > 64 MB anzuzeigen, d.h. es werden nur die unteren
                   64 MB verwendet.
  /X               Verwendet nicht die INT 15 Funktion E820h um Speicher
                   > 64 MB anzuzeigen.
  /X2MAX32         Begrenzt den XMS 2.0 frei/erhältlich Speicherbericht
                   auf 32 MB - 1 KB.
  /?               Zeigt die Hilfe.

Kommentar:

  HIMEMX erfordert eine 80386+ CPU. JEMM386 kann erst nach diesem
  Treiber geladen werden. Laden Sie HIMEMX mit "DEVICE=HIMEMX.EXE" in
  der CONFIG.SYS (obwohl es eine .exe-Datei ist)!

Beispiel:

  IN DER CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himemx.exe
    device=c:\fdos\bin\jemm386.exe
