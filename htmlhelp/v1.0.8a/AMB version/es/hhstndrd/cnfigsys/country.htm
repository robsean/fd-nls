Comando: country

  COUNTRY establece el comportamiento internacional de DOS.
  COUNTRY es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  country=nnn[,[mmm]][,[unidad][ruta]nombre_archivo]
          unidad          La letra de la unidad, por ejemplo C:
          ruta            El directorio, por ejemplo \ejemplo\
          nombre_archivo  El nombre de archivo de un archivo de 
                          información del país. Si no se especifica, el
                          archivo 'COUNTRY.SYS' es el que se utilizará.

Opciones:

  nnn    El código de país, como se muestra en la tabla siguiente 
         (001-EE.UU.).
  mmm    La página de códigos, como e muestra en la tabla siguiente (437
         es por defecto, 850 se actualiza el formulario, 1252 para 
         Windows). Si no se especifica, se elegirá automáticamente una
         página de códigos.

  Códigos de país
  En la tabla:

    * HH significa una hora de 24 horas
    * hh una hora de 12 horas.
    * X significa que AM y PM se añaden

Comentatios:

  COUNTRY habilita/establece las características internacionales de DOS
  seleccionando un código de país y (sólo con el núcleo inestable) 
  opcionalmente también una página de códigos y un archivo de país. El
  núcleo estable sólo utiliza el valor nnn. Para obtener soporte completo,
  utilice el kernel inestable/en desarrollo. Usted también necesita un 
  COUNTRY.SYS para el segundo, mientras que el primero ignora el nombre de
  archivo y sólo establece formatos de fecha/hora/número, utilizando 
  valores compilados en el núcleo para algunos países comunes. No 
  establece orden y las tablas de traducción en mayúsculas y minúsculas.
  NLSFUNC sólo puede utilizarse con el núcleo inestable.
  COUNTRY es un comando interno del kernel.sys y no necesita otro archivo
  para poder ejecutarse.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himemx.exe
    device=c:\fdos\bin\jemm386.exe
    dos=high,umb
    country=001,437,c:\fdos\bin\country.sys
