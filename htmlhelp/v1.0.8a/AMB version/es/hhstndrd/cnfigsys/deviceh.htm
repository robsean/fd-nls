Comando: devicehigh

  DEVICEHIGH carga un controlador en memoria alta, mientras que DEVICE 
  carga el controlador en memoria baja o convencional.
  DEVICEHIGH es un comando CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  devicehigh=[unidad][ruta]nombre_archivo [opciones]
             unidad         La letra de la unidad, por ejemplo C:
             ruta           El directorio por ejemplo \ejemplo\
             nombre_archivo El controlador (archivo) que se va a cargar,
                            por ejemplo himemx.exe

Opciones:

  opciones  Estas son opciones para el controlador del programa, no
            al comando 'device'.

Comentarios:

  DEVICEHIGH carga el controlador de dispositivo especificado por la
  unidad:\ruta\nombre_archivo en memoria alta [ABOVE 640KB] (en su defecto
  se cargará en memoria convencional). Las opciones son para el propio
  controlador; consulta la documentación que venía con su dispositivo
  particular para las opciones compatibles y su uso.
  A medida que DEVICEHIGH carga el controlador en una memoria alta, sigue
  teniendo mas espacio libre en memoria baja o convencional para otros
  controladores o para programas. Para cargar un controlador o un programa
  en una memoria alta, primero tienes que cargar un controlador como 
  (HIMEM), HIMEMX o JEMMEX en CONFIG.SYS / FDCONFIG.SYS.
  DEVICEHIGH es un comando interno del kernel.sys y no necesita otro 
  archivo para poder trabajar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    DEVICE=C:\FDOS\BIN\XMGR.SYS /N128 /B
    DEVICEHIGH=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
  DENTRO DE UN MENú:
    23?DEVICEHIGH=C:\FDOS\BIN\XMGR.SYS /N128 /B
    23?DEVICEHIGH=C:\FDOS\DRIVERS\UIDE.SYS /S125 /D:MYDVD
