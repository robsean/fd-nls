Comando: menudefault

  MENUDEFAULT establece una opción predeterminada de tiempo retrasado para
  un menú config.sys / fdconfig.sys.
  MENUDEFAULT es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  menudefault=opción_por_defecto,retraso

Opciones:

  retraso             tiempo de retardo en segundos
  opción_por_defecto  utiliza la opción estándar si el tiempo de retardo
                      ha terminado

Comentarios:

  MENUDEFAULT establece una opción predeterminada de tiempo retrasado para
  un menú config.sys / fdconfig.sys: Si no se ha pulsado ninguna tecla
  durante los segundos de 'retardo', se activa la opción predeterminada.
  Al pulsar una tecla se detiene la cuenta atrás, el núcleo esperará
  infinitamente a que el usuario realice una selección. Tenga en cuenta que
  algunas máquinas virtuales como Bochs pueden haber roto (demasiado
  rápido) el tiempo.
  MENUDEFAULT es un comando interno de kernel.sys y no necesita ningún
  otro archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    menucolor 15,1
    MENU Elija sabiamente:
    MENU
    MENU 0)    controladores normales (himem y emm386)
    MENU 1)    controladores alternativos (himemx y jemm386)
    MENUDEFAULT=0,5
    0?DEVICE=A:\himem.exe
    1?DEVICE=A:\himemx.exe
    0?DEVICE=A:\emm386.exe
    1?DEVICE=A:\jemm386.exe
  El menú esperará 5 segundos. Si no se pulsa ninguna tecla en este
  tiempo, la opción 0 se selecciona automáticamente. Menucolor 15,1 cambia
  el color a texto blanco sobre fondo azul.
    0 elige himem.exe y emm386.exe, 
    1 elige himem.exe y jemm386.exe
