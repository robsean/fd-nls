Comando: switches

  SWITCHES ajusta el comportamiento de procesamiento del tiempo de 
  arranque.
  SWITCHES es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  switches=/E[:xxx] /F /K /N /W

Opciones:

  /F   omite la comprobación de retraso para la pulsación de tecla F5/F8
       antes de procesar config.sys [equivalente a SYS CONFIG 
       skipconfigseconds=0]
       F5 y F8 solo se procesan si se presionan antes de que se inicie
       DOS, pero después de configurar el teclado, es fácil pasará por
       alto el momento correcto.
  /N   deshabilita el soporte F5/F8 [equivalente a la configuración del
       núcleo (SYS CONFIG, ejecute SYS CONFIG /? para explicaciones)
       skipconfigseconds=-1] F5 (omitir configuración) y F8 (solo paso
       config.sys) se ignoran.
  /K   obliga a tratar el teclado como un teclado de 86 teclas, no como
       una teclado 102/105. Puede ser útil con BIOS o controladores que
       no tienen o tienen un manejo roto para teclados de 102/105 teclas.
  /E   permite mover EBDA (Area de datos de BIOS extendida), opcionalmente
       se puede especificar un tamaño en kilobytes [xxx, en un rango de 
       48-1024]
  /W   NO es compatible con FreeDOS. Esta opción en MS-DOS establecería
       un indicador para Windows 3.0 para omitir la carga de wina20.386
       desde el directorio raíz.

Comentarios:

  SWITCHES es un comando interno de kernel.sys y no necesita ningún otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    switches=/E[:48] /F /K /N /W
