Comando: idlehalt

  IDLEHALT activa la funcionalidad integrada de ahorro de energía del
  núcleo si n no es 0.
  IDLEHALT es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  idlehalt=n
    Donde n puede ser -1, 0, 1 o superior (predeterminado 0)

Opciones:

  - ninguna -

Comentarios:

  IDLEHALT activa la funcionadidad de ahorro de energía del núcleo
  incorporado si n no es 0. El valor -1 habilita todos los ganchos, 1
  habilita solo los ganchos "seguros", la CPU se detiene solo si el núcleo
  está esperando la entrada del dispositivo CON con un char.
  Los ganchos adicionales para n=-1 y n>0 dependen de la versión del
  núcleo: Además de los ganchos seguros, se pueden activar otros ganchos,
  por ejemplo uno para int 0x2f, ax=0x1680 "intervalo de tiempo de
  liberación"
  Nota: En casos excepcionales, entrar o salir del modo HLT (lo que provoca
  grandes cambios en el consumo de energía de la CPU) puede causar fallos
  si las fuentes de alimentación son baratas o las placas base no se pueden
  filtrar adecuadamente los transitorios. Hacer un overlocking de todo el
  sistema puede ayudar.
  Linux siempre hace algunas HLT en el momento del arranque, para forzar un
  bloqueo anticipado de los sistemas con errores (Arranque sin-hlt para
  deshabilitar HLT). P90 puede tener buggy HLT?
  www.tavi.co.uk/ps2pages/ohland/halt.html
  IDLEHALT es un comando interno de kernel.sys y no necesita ningún otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    idlehalt=1
