Comando: files

  FILES le dice a FreeDOS cuántos archivos debería poder abrir de forma
  simultánea.
  FILES es un comando CONFIG.SYS / FDCONFIG.SYS command.

Sintaxis:

  files=n
    Donde n es el número de archivos, en el rango de 8-255. La 
    configuración predeterminada es 8, pero un buen número para la mayoría
    de los sistemas modernos es 20-40. Algunos programas pueden necesitar
    una configuración más alta para funcionar correctamente; dichos
    programas generalmente le darán el valor que se necesita en su
    documentación.

Opciones:

  - ninguna -

Comentarios:

  FILES especifica cuántos archivos se pueden abrir a la vez (reserva la
  memoria necesaria para admitir la apertura de tantos archivos).
  Nota: existen otras restricciones, por lo que es posible que un programa
  determinado no pueda abrir esta cantidad. un buen número es 20, aunque
  algunos programas sugieren / requieren 30, 40 o incluso 255.
  FILES es un comando interno de kernel.sys y no necesita ningún otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    files=20
