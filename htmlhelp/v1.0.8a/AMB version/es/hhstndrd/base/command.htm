Comando: command (command.com o FreeCom)

  COMMAND inicia una nueva copia del shell de comandos de FreeDOS.

Sintaxis:

  COMMAND [[unidad]ruta] [dispositivo] [/E:nnnnn] [/L:nnnn] [/U:nnn]
          [/P] [/MSG] [/LOW] [/Y [/[C|K] comando]]
  COMMAND [/?]
          [unidad]ruta   Especifica el directorio que contiene
                         COMMAND.COM, por ejemplo C:\ejemplo\
          unidad         La letra de unidad, por ejemplo C:
          ruta           El directorio, \ejemplo\
          dispositivo    Especifica el dispositivo a utilizar para la
                         entrada y salida de comandos.

Opciones:

  /E:nnnnn    Establece el tamaño del entorno inicial en nnnnn bytes.
              (nnnnn debe estar entre 256 y 32,768).
  /L:nnnn     Especifica la longitud de los búferes internos (también
              requiere /P) (nnnn debe estar entre 128 y 1,024).
  /U:nnn      Especifica la longitud del búfer de entrada (también
              requiere /P) (nnn debe estar entre 128 y 255).
  /P          Hace que el nuevo shell de comando sea permanente (no
              puede salir).
  /MSG        Almacena todos los mansajes de error en la memoria
              (también requiere /P).
  /LOW        Fuerza al shell de comandos a mantener sus datos residentes
              en memoria baja.
  /Y          Pasos a través del programa por lotes especificados por
              /C o.
  /C comando  Ejecuta el comando especificado y devuelve.
  /K comando  Ejecuta el comando especificado y continúa.
  /?          Muestra la ayuda.

Comentarios:

  [unidad]ruta  La unidad y la ruta donde el shell buscará la parte
                transitoria del programa. Esto generalmente solo es
                necesario para configurar el COMSPEC.
  dispositivo   Un dispositivo alternativo para todas las E/S.
  /e:nnnnn      El tamaño del entorno, en bytes, en el rango 160-32768.
                Este número se redondeará a los 16 bytes más cercanos.
                El valor predeterminados es 256.
  /p            Hace que el shell sea permanente, por lo que el comando
                EXIT no sale del shell.
  /c {cadena}   Ejecuta el comando en {cadena}, luego sale.
  /msg          Carga cualquier mensaje de error que pueda almacenarse
                en el disco dentro de la memoria.
  COMMAND (COMMAND.COM) también se llama FreeCOM. Contiene muchos 
  comandos internos que no necesitan otro archivo para funcionar. Puede
  encontrar la lista de comandos internos here.

Ejemplos:

  C:\comando /E:1024 /L:1024 /U:128 /P
  comando /?
