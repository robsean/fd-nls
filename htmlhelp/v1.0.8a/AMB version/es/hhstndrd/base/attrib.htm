Comando: attrib

  ATTRIB muestra o cambia los atributos del archivo.

Sintaxis:

  ATTRIB { opciones | [unidad][ruta]nombre_archivo | /@[lista] }
         unidad          La letra de unidad, por ejemplo C:
         ruta            El directorio, por ejemplo \ejemplo\
         nombre_archivo  El archivo, por ejemplo texto.exe

Opciones:

  +A  Establece el atributo de archivo.
  -A  Borra el atributo de archivo.
  +H  Establece el atributo oculto.
  -H  Borra el atributo oculto.
  +R  Establece el atributo de solo lectura.
  -R  Borra el atributo de solo lectura.
  +S  Establece el atributo del sistema.
  -S  Borra el atributo de sistema.
  /@  Procesar archivos, listados en el archivo especificado [o en stdin]
  /D  Procesar nombres de directorio para argumentos con comodines.
  /S  Procesar archivos en todos los directorios en la rutas
      especificadas.
  /?  Muestra la ayuda.

Comentarios:

  Si no se especifica ningún archivo, los atributos para todos los
  archivos en el directorio actual se cambian (o simplemente se muestran
  si no se utiliza ninguno de los especificadores anteriores). El 
  atributo de archivo (+A) se establece cuando se modifica un archivo.
  Las utilidades de respaldo pueden borrar el atributo para indicar que
  el archivo ha sido respaldado. Si un archivo tiene el indicador oculto
  (+H), no se muestra si escribe "dir". Si un archivo tiene el indicador
  de solo lectura (+R), puede leerse pero no modificarse. Si un archivo
  tiene el indicador del sistema (+S), está especialmente protegido, no
  se puede eliminará ni sobrescribir. ATTRIB tiene los siguientes 
  EXITCODES (ERRORLEVEL):
    0  finalización exitosa; /? opción utilizada
    1  error al cambiar los atributos de los archivos
    2  no se encontraron archivos coincidentes para los objetivos
    3  archivos abiertos de error, señalados en /@ opción(es)
    4  error(es) en la línea de comandos

Ejemplos:

  attrib archivo -rhs
  attrib +a -r dir1 dir2*.dat /s
  attrib -hs/sd /@lista.txt *.*
