Comando: append

  APPEND permite a los programas abrir archivos de datos en directorios
  específicos como si los archivos estuvieran en el directorio actual.

Sintaxis:

  APPEND [[unidad]ruta[;...]] [/X[:ON|:OFF]] [/PATH:ON|/PATH:OFF] [/E]
  APPEND ;
  APPEND [/?]
         unidad  Letra de unidad, por ejemplo C:
         ruta    Directorio para agregar, \ejemplo

Opciones:

  /X[:ON]    Extiende APPEND a búsquedas y ejecución de comandos.
  /X:OFF     Aplica APPEND solo a solicitudes para abrir archivos. Por
             defecto es /X:OFF
  /PATH:ON   Buscar en los directorios anexados las solicitudes de 
             archivos que ya incluyen una ruta. Esta es la configuración
             predeterminada.
  /PATH:OFF  Desactiva /PATH:ON.
  /E         Almacena la lista de directorios adjunta en el entorno. /E
             solo se puede usar en la primera invocación de APPEND. No
             puede incluir ninguna ruta en la misma línea de comandos
             que /E.
  APPEND ;   borra la lista de directorios adjuntos.
  APPEND     sin parámetros muestra la lista de directorios adjuntos.
  /?         Muestra la ayuda.

Comentarios:

  No use APPEND con Windows o extensores de 32 bits. APPEND se instala
  como un comando interno después de su primera ejecución. Las 
  invocaciones segundas y sucesivas DEBEN excluir la ruta del archivo y
  la extensión de comando APPEND. APPEND admite NLS (soporte de idioma
  nacional).

Ejemplos:

    Primer uso:
  C:\command\append.exe D:\Data\df1.dat
    Uso perfectamente legítimo del comando Append para la primera
    invocación de append.
    Segundo uso incorrecto:
  C:\command\append.exe D:\Data\df2.dat
    Segundo uso no válido. Resulta en el nuevo apéndice cargado por
    command.com viéndose en la memoria y saliendo con un error.
    Segundo uso válido:
  append D:\Data\df2.dat
    Válido segundo uso. Command.com mira su lista de comandos internos,
    ve anexar y lo ejecuta pasando el resto de la línea de comandos.
