Mise en réseau de FreeDOS - PKT2ODI

  En tant qu'alternatif à ODIPKT.COM qui est plus communément utilisé,
  certaines personnes peuvent aussi vouloir essayer PKT2ODI.EXE de
  Caldera comme un convertisseur pilote de paquets vers ODI. Mais nous
  devons garder à l'esprit que la licence de WebSpyder de Novell (qui
  s'applique aussi à PKT2ODI.EXE) ne permet que d'utiliser le logiciel
  dans un but d'évaluation.
  Pour les utilisateurs commerciaux, la durée d'évaluation et limitée
  à 90 jours. Pour les utilisateurs non commerciaux, l'évaluation est
  permise pour "une durée raisonnable". (Jusqu'à quel point peut-il être
  raisonnable d'évaluer ce logiciel au XXIe siècle?)
  En tout cas, PKT2ODI.EXE devra être démarré dans AUTOEXEC.BAT avec une
  commande comme celle-ci :
    LH C:\NETWORK\NWCLIENT\PKT2ODI.EXE /B:1 /I:69
  L'option "/B" indique à PKT2ODI.EXE quelle carte logicielle il doit
  utiliser. Comme expliqué plus haut, nous devons choisir la carte qui
  utilise la trame ETHERNET II. À la différence de ODIPKT, PKT2ODI
  commence à compter avec "1", donc si NET.CFG ressemble à cet exemple :
    --- NET.CFG ---
      Link Driver 3C574
        FRAME ETHERNET_II
        FRAME ETHERNET_802.2
        FRAME ETHERNET_802.3
        FRAME ETHERNET_SNAP
  nous devons utiliser le paramètre
    PKT2ODI.EXE /B:1
  pour définir l'utilisation de la première carte (celle avec
  ETHERNET_II). Rappelez-vous que nous devons utiliser la carte avec
  Ethernet II, sinon le convertisseur ne pourra pas communiquer avec
  le pilote et se plaindra de ne pas trouver le MLID (Multiple Link
  Interface Driver).

  Le second paramètre, le "/I" définit l'interruption logicielle
  (vecteur) utilisé par le pilote. Nous savons déjà que cela doit
  être 0x60, mais malheureusement cela ne fonctionnera pas avec
  PKT2ODI. Le programme n'accepte pas les vecteurs d'interruption
  0x60 à 0x68. Nous pouvons utiliser le vecteur 0x69 avec le paramètre :
    PKT2ODI.EXE /I:69
  L'utilisation d'un vecteur d'interruption inhabituelle comme 0x69
  peut être un problème pour certaines applications TCP/IP. Elles
  doivent être configurées pour utiliser ce vecteur. Certains
  programmes, comme le serveur web SIOUX, peut ne pas fonctionner
  correctement si le vecteur n'est pas 0x60.
  C'est tout. Après avoir démarré la LSL (Link Support Layer,
  c'est-à-dire la couche de support de liaison), le MLID (Multiple
  Link Interface Driver), le convertisseur et le pilote de paquets
  devraient maintenant être opérationnels.
