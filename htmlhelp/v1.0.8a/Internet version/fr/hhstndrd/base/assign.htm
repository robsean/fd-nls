Commande : assign

  ASSIGN réaffecte les opérations de disque d'un lecteur à un autre.

Syntaxe :

  ASSIGN [/SHADOW] [/STATUS] [/?]
  ASSIGN [/SHADOW] [/M] [/U] [{lecteur1[:]=lecteur2[:]}]
         lecteur1:  Lettre de lecteur devant être remplacée, par ex. C:
         lecteur2:  Lettre de lecteur vers laquelle pointer, par ex. D:

Options :

  /SHADOW  Rend la présence d'ASSIGN dans la mémoire difficile à
           détecter. Les utilisations suivantes d'ASSIGN émettront un
           avertissement si l'option /shadow n'est pas utilisée (mais
           exécuteront tout de même la commande).
  /STATUS  Affiche toutes les lettres de lecteur attribuées.
           Si ASSIGN n'est pas déjà chargé, un message vous en informe
           mais ASSIGN ne sera pas chargé.
  /M       Tente d'utiliser la mémoire haute au lieu de la mémoire
           conventionnelle pour stocker ASSIGN.
  /U       Désinstalle un ASSIGN déjà installé.
  /?       Affiche l'aide.

Commentaires :

  Les caractères ':' sont optionnels, entre les lettres de lecteur et
  le  signe '=' aucune espace n'est permise.
  Si aucun lecteur n'est nommé et que /status ou /u ne sont pas
  utilisés, alors assign est réinstallé (effaçant toutes les
  réaffectations).
  ASSIGN a les CODES DE SORTIE (ERRORLEVEL) suivants :
    0  succès / aide
    3  annulé par l'utilisateur (par ex. Ctrl-C)
    4  erreur de syntaxe
  N'utilisez pas assign en combinaison avec les commandes suivantes :
    diskcomp
    diskcopy
    format
    label
    print
    restore
    subst

Exemples :

  assign B:=A:
  assign B=A

Voir également :

  join
  subst
  swsubst
  truename
