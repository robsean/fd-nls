Commande : printer.sys

  PRINTER.SYS fournit des capacités de gestion de pages de code à un
  pilote d'imprimante. PRINTER.SYS doit être chargé dans CONFIG.SYS /
  FDCONFIG.SYS.
  Lorsque FreeDOS fonctionne déjà, vous pouvez charger PRINTER.SYS par
  la suite avec DEVLOAD.

Syntaxe :

  printer [nompériph][:]=([typematériel],[{n|(n1,[n2,]...)}],[m])

Options :

  nompériph     Nom du périphérique du port d'imprimante qui doit être
                étendu, tel que PRN or LPTx. Les deux-points sont
                optionnels.
  typematériel  Identificateur du type d'adaptateur matériel d'affichage.
                Il peut aussi déterminer le nombre de sous-polices à
                utiliser.
                Il n'existe pas encore de types pris en charge.
  n, n1, n2,..  Numéros pour les numéros de page de code des pages de
                code du  matériel.
  m             Le nombre de tampons que vous souhaitez spécifier pour
                préparer les pages de code (la valeur par défaut
                est de 1)

Commentaires :

  Ce pilote n'a pas encore de forme binaire, car aucune gestion
  d'imprimante spécifique n'a encore été ajoutée (vous pouvez compiler
  à la fois PRINTER.SYS et DISPLAY.SYS depuis les sources de DISPLAY).

Exemples :

  Lorsque PRINTER.SYS aura un support de matériel, vous pourrez entrer :
    PRINTER PRN=(IBMPro,437,1)
  Vous spécifiez que vous avez une imprimante compatible IBM Proprinter,
  avec une page de code matérielle 437, et demandez seulement 1 tampon
  dans le but de minimiser l'empreinte mémoire résidente.
