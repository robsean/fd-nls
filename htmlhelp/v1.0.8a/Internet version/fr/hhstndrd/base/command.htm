Commande : command (command.com ou FreeCom)

  COMMAND démarre une nouvelle copie du shell de commandes de FreeDOS.

Syntaxe :

  COMMAND [[lecteur]chemin] [appareil] [/E:nnnnn] [/L:nnnn] [/U:nnn]
          [/P] [/MSG] [/LOW] [/Y [/[C|K] commande]]
  COMMAND [/?]
          [lecteur]chemin Spécifie le dossier contenant COMMAND.COM,
                       par exemple C:\exemple\
          lecteur      La lettre du lecteur, par exemple C:
          chemin       Le répertoire, par exemple \exemple\
          appareil     Spécifie les périphériques à utiliser pour les
                       entrées et sorties de commandes.

Options:

  /E:nnnnn    Définit la taille initiale de l'environnement à nnnnn
              octets. (nnnnn devrait être entre 256 et 32,768).
  /L:nnnn     Spécifie la taille du tampon interne (nécessite /P
              également). (nnnn devrait être entre 128 et 1,024).
  /U:nnn      Spécifie la taille du tampon d'entrée (nécessite /P
              également). (nnn devrait être entre 128 et 255).
  /P          Rend le nouveau shell permanent (impossible d'en sortir).
  /MSG        Stocke tous les messages d'erreur dans la mémoire
              (nécessite /P).
  /LOW        Force le shell de commande à garder ses données résidentes
              en mémoire basse.
  /Y          Parcourt le programme batch spécifié par /C o.
  /C command  Exécute la commande spécifiée et retourne.
  /K command  Exécute la commande spécifiée et continue.
  /?          Affiche l'aide.

Commentaires :

  [lecteur]che Le lecteur et le chemin où le shell doit chercher la
               partie transitoire du programme. Cela n'est généralement
               nécessaire que pour définir le COMSPEC.
  appareil     Un périphérique alternatif pour toutes les e/s.
  /e:nnnnn     La taille en octets de l'environnement, entre 160 et
               32768. Ce chiffre sera arrondi aux 16 octets les plus
               proches. La valeur par défaut est 256.
  /p           Rend le shell permanent, donc la commande EXIT ne sort
               pas du shell.
  /c {chaîne}  Exécute la commande dans {chaîne}, puis quitte.
  /msg         Charge tous les messages d'erreur qui pourraient être
               stockés sur le disque dans la mémoire.
  COMMAND (COMMAND.COM) est également nommé FreeCom. Il contient de
  nombreuses commandes internes qui ne nécessitent aucun autre fichier
  pour fonctionner.
  Vous pouvez trouver la liste des commandes internes ici.

Exemples :

  C:\command /E:1024 /L:1024 /U:128 /P
  command /?
