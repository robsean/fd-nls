Commande : swsubst

  SWSUBST combine subst avec join avec des options améliorées.
  SWSUBST remplace un chemin par une lettre de lecteur pour que vous
  ayez accès à un chemin par une lettre de lecteur.

Syntaxe :

  swsubst [option] [commande]
  Si swsubst est utilisée sans spécifier une commande, alors la table
  CDS est affichée.

Options :

  /#     Affiche les statistiques des lecteurs JOINts dans le tableau
         CDS.
  /!     Comme /#; initialise l'indicateur JOIN, s'il est différent.
  /f     Affiche les caractères non imprimables en hexadécimal.
  /a     Affiche le tableau CDS après la commande.
  /k     Crée les chemins mentionnés dans la commande.
  /t     Comme /k, dans SUBST vérifie seulement, si le chemin existe.
  /_     Vide tous les attributs de la table CDS.
  /o=#   Utilise l'offset de la barre oblique inversée, ne le calcule
         pas.
  /x=??  Liste des lettres de lecteur à ignorer pendant une recherche
         ::label:
  Commandes :
  -                      Initialise tous les lecteurs JOINts et SUBST.
  --                     Initialise tous les lecteurs non mis en réseau.
  dr [-]chemin           Similaire à %s dr [-]chemin.
  /u dr [-]chemin        Substitue dr pour pointer vers le chemin.
  /j dr [-]chemin        Rejoint dr au chemin.
  /d dr | dr /d | dr -   Initialise le lecteur dr (et casse SUBST/JOIN).
  /s dr1 dr2             Échange dr1 et dr2.
  /n [dr1 dr2]           Échange les numéros d'unité du même pilote de
                         périphérique (Std.: disquettes).
  /w dr                  Définit ERRORLEVEL sur la lettre du lecteur.
  /m [-]chemin           Crée un chemin incluant tous les principaux
                         répertoires au-dessus.
  /c [-]chemin           Comme /m, exécute ensuite chdir pour le
                         répertoire.
  dr:[+-=]{off,on,phys,net,join,subst,hidden}  Définit directement les
                                               indicateurs du lecteur.
  /q dr:[+-]{phys,net,join,subst,hidden}       Interroge les indicateurs
                                               du lecteur.

Commentaires :

  Les entités standards de DOS émulant SWSUBST, SUBST and JOIN, ont la
  même ligne de commande que les DOS. Mais elles diffèrent sur les
  points suivants :
    - Le chemin spécifié sera créé, avec le chemin complet y menant,
      par exemple "SUBST c: d:/blah/humpf" créé le chemin
      "d:\blah\humpf", que d:\blah existe ou non.
    - Vous êtes autorisé à rompre les relations à la volée, ce qui veut
      dire :
      quelque chose de déjà sujet à SUBST peut être à nouveau sujet à
      un autre SUBST ou JOIN sans notification ou avertissement
      supplémentaire.
    - Vous êtes autorisé à traiter un lecteur avec lui-même, par
      exemple : "JOIN a: a:\blah"
    - Le vidage du tableau est différent, SWSUBST affiche
      "X: => Y:\CHEMIN" au lieu de "X: is joined to Y:\CHEMIN".
    - Vous ne pouvez pas renommer les exécutables SUBST and JOIN.
    - La version de DOS ne sera pas testée, sauf si c'est nécessaire
      pour ajuster les tailles des différentes structures de données.
  Si vous avez besoin de SUBST ou JOIN exécutez simplement :
  "copy SWSUBST.EXE JOIN.EXE ou SUBST.EXE".
  Un ERRORLEVEL entre 65 (ASCII 'A') et 90 (ASCII 'Z') indique OK
  et la lettre du lecteur lui-même ; alors qu'un ERRORLEVEL de zéro
  indique qu'il n'existe pas un tel lecteur ; et les autres ERRORLEVELS
  indiquent une erreur.

Exemples :

  swsubst b: c:\blah\humpf
