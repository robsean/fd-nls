Commande : display

  DISPLAY charge les capacités de gestion de page de code de la console.

Syntaxe :

DISPLAY nompériph[:]=(TypeMatériel,[n|(n1[,n2]...)],[m|(m,p)])
        [/V] [/C] [/NOHI]

Options:

  nompériph     Un nom de périphérique valide (ignoré dans cette
                version). Utilisez ce que vous souhaitez (tel que CON).
  TypeMatériel  Une étiquette de type de matériel supporté (tel que EGA
                ou VGA)
  n             page de code matériel par défaut (pour une page de code
                matérielle). Ce sera normalement 437.
  n1, n2,...    Pages de code matérielles, si plus d'une est utilisée
                (généralement vous devrez utiliser des pilotes tiers
                pour l'arabe/l'hébreu)
  m             Nombre de tampons pour les pages de code préparées
                (1 par déf).
  p             Paramètre additionnel spécifique au matériel.
  /C            ExClusif : ne charge pas si DISPLAY est déjà présent.
  /NOHI         Désactive l'utilisation de la mémoire XML pour les
                pages de code préparées.
  /V            Verbeux : informations étendues.

Commentaires :

Matériel pris en charge :
  VGA     Adaptateurs VGA. 3 sous polices seront utilisées dans tous
          les cas.
  EGA     Détermine automatiquement le nombre de sous polices à utiliser.
  EGA 14  Force l'utilisation de deux sous polices (8x8 et 8x14).
  EGA 8   Force l'utilisation de seulement une sous police (8x8).
  LCD     Adaptateurs LCD (identique à EGA 8).

          Pour tous les adaptateurs ci-dessus, le paramètre p tentera de
          forcer le nombre de sous polices à utiliser (1=8x8, 2=8x8,
          8x14, 3=8x8, 8x14, 8x16) dès que le matériel l'autorise.
  CGA     Adaptateur CGA (FreeDOS GRAFTABL). Le paramètre p est ignoré.

  DISPLAY ne produit aucun CODE DE SORTIE, mais offre un jeu d'erreurs
  via la fonction int2Fh/AD03h pour chaque fonction. Vous devriez le
  vérifier à la suite de chaque opération non couronnée de succès.

  Code d'erreur Opération    Signification
  --------------------------------------------------------------
  1            All           Fonction inconnue
  26           Select        La page de code n'était pas préparée
  26           Query         Aucune page de code n'était déjà
                             sélectionnée
  27           Select        KEYB a échoué à changer la page de code
  27           Query         Erreur du périphérique
  27           Prepare       La page de code n'a pas été trouvée dans
                             le fichier
  29           Select        Erreur du périphérique
  29           Prepare       Erreur du périphérique ou erreur XMS
  31           Prepare       Fichier source endommagé ou erreur XMS
  31           End Prepare   Aucun Start prepare trouvé
  --------------------------------------------------------------

Exemples :

  DISPLAY CON=(VGA,437,1)
    Adaptateur VGA dont la page de code BIOS est 437. 1 tampon pour la
    page de code logiciel est utilisé. C'est le standard pour la
    plupart des cas.

  DISPLAY CON=(EGA,,2)
    L'adaptateur graphique est EGA ou mieux, mais n'est pas connu. La
    page de code par défaut du BIOS est aussi inconnue. Deux tampons
    sont nécessaires.
