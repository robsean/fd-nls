Commande : attrib

  ATTRIB affiche ou modifie les attributs de fichiers.

Syntaxe :

  ATTRIB { options | [lecteur][chemin]nomdefichier | /@[list] }
         lecteur     La lettre du lecteur, par ex. C:
         chemin      Le répertoire, par ex. \exemple\
         nomdefich   Le fichier, par ex. test.exe

Options :

  +A  Définit l'attribut Archive.
  -A  Efface l'attribut Archive.
  +H  Définit l'attribut Caché.
  -H  Efface l'attribut Caché.
  +R  Définit l'attribut Lecture-seule.
  -R  Efface l'attribut Lecture-seule.
  +S  Définit l'attribut Système.
  -S  Efface l'attribut Système.
  /@  Traite les fichiers listés dans le fichier spécifié [ou dans
      stdin].
  /D  Traite les noms de dossiers par rapport aux arguments avec des
      jokers.
  /S  Traite les fichiers dans tous les dossiers dans les chemins
      spécifiés.
  /?  Affiche l'aide.

Commentaires :

  Si aucun fichier n'est spécifié, alors les attributs de tous les
  fichiers dans le dossier courant seront changés (ou simplement
  affichés si aucun des spécificateurs ci-dessus n'est utilisé.)
  L'attribut archive (+A) est défini lorsqu'un fichier est modifié. Les
  utilitaires de sauvegarde peuvent effacer l'attribut pour indiquer que
  le fichier a été sauvegardé.
  Si un fichier a l'attribut caché (+H) il n'est pas montré quand vous
  tapez dir.
  Si un fichier a l'attribut lecture seule (+R) il ne peut être que lu.
  Si un fichier a l'attribut système (+S), il est spécialement protégé,
  par ex. il ne peut être effacé ou écrasé.
  ATTRIB a les CODES DE SORTIE EXITCODES (ERRORLEVEL) suivants :
    0  achèvement réussi ; option /? utilisée.
    1  erreur de modification des attributs des fichiers.
    2  aucun fichier correspondant trouvé par rapport aux cibles.
    3  erreur d'ouverture des fichiers indiqués dans l'option /@.
    4  erreur sur la ligne de commande.

Exemples :

  attrib file -rhs
  attrib +a -r dir1 dir2*.dat /s
  attrib -hs/sd /@list.txt *.*
