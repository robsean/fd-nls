Commande : emm386

  EMM386 est le gestionnaire de mémoire étendue.
  EMM386 doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.
  EMM386 est périmé. Vous devriez utiliser HIMEMX / JEMM386
  ou JEMMEX à la place.

Syntaxe :

  DEVICE=[lecteur][chemin][EMM386.EXE] [options]
         lecteur  La lettre du lecteur, par exemple C:
         chemin   Le répertoire, par exemple \exemple\
  ou lancez-le depuis la ligne de commande :
  [lecteur][chemin][emm386.exe] [options]

Options :

  ALTBOOT       Accrocher l'interruption clavier pour le traitement du
                redémarrage.
  EMM=#####     Réserver jusqu'à #####Ko pour la mémoire EMS/VCPI
                seulement.
  FRAME=E000    Sélectionner le pageframe voulu pour EMS.
  I=hhhh:hhhh   Utiliser la mémoire qui réservée pour certains
                périphériques. Par exemple :
                  I=A000-AFFF (graphiques VGA)
                  I=B000-B7FF (graphiques Hercules)
                Utiliser I=hhhh:hhhh pour forcer UMB sur d'autres
                endroits.
  I=TEST        Tester les localisations ROM pour l'inclusion UMB.
  MAX=#####     Mémoire maximale disponible pour VCPI en Ko ; également
                EMS si inférieur à 32 Mo.
  MEMCHECK      Accéder à l'espace d'adressage complet de 4Go sans RAM
                (MMIO).
  NOALTBOOT     Ne pas accrocher l'interruption clavier pour le
                traitement du redémarrage (valeur par défaut).
  NOCHECK       Interdit l'accès à l'espace d'adressage sans RAM (MMIO).
  NODISABLEA20  Ne pas autoriser EMM386 à désactiver A20.
  NOEMS         Ne pas activer la mémoire étendue. Utiliser NOEMS pour
                obtenir plus d'UMB libres si vous n'avez pas besoin
                d'EMS.
  NOVCPI        Ne pas utiliser VCPI (doit être utilisé avec NOEMS).
  NOVDS         Désactiver les services virtuels DMA.
  SB            Mode de compatibilité avec le pilote SoundBlaster.
  VDS           Active les services virtuels DMA (valeur par défaut).
  /VERBOSE      Afficher une sortie détaillée lorsque emm386 est chargé.
  X=hhhh:hhhh   Définir une plage de mémoire pour les périphériques
                mappés en mémoire.
                Par exemple X=D000-D800 X=hhhh:hhhh peut être nécessaire
                par exemple pour certains réseaux.
  X=TEST        Tester les emplacements de la ROM pour déterminer s'ils
                sont exclus de l'UMB.
  /?            Afficher l'aide.

Commentaires :

  EMM386 nécessite un processeur 80386+. HIMEM devrait être chargé avant.
  Chargez EMM386 comme DEVICE=EMM386.EXE dans CONFIG.SYS (bien que ce
  soit un fichier ".exe" !)
  EMM386 doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.

Exemples :

  IN CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\himem.exe
    device=c:\fdos\bin\emm386.exe
