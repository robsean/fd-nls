Commande : fdxms.sys

  FDXMS.SYS est un pilote de mémoire étendue pour FreeDOS.
  FDXMS.SYS est destiné aux ordinateurs 386+. Fdxms286 est destiné aux
            ordinateurs 286.
  FDXMS.SYS doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.
  FDXMS.SYS est périmé. Vous devriez utiliser HIMEMX ou JEMMEX
  à la place.

Syntaxe :

  DEVICE=[lecteur][chemin]\fdxms.sys [options]
         lecteur  La lettre du lecteur, par exemple C:
         chemin   Le répertoire, par exemple \exemple\

Options :

  NUMHANDLES=  Définit le nombre de descripteurs XMS. La valeur par
               défaut est 32. La valeur minimale est deux (XMS n'est pas
               utile sans aucun handle !), la valeur maximale est
               actuellement 1024. Chaque descripteur prend 5 octets
               (FDXMS.SYS) ou 9 (FDXXMS.SYS).
  BIOS         Fait que FDXMS appelle toujours le BIOS via INT15 au lieu
               d'utiliser sa propre commutation de mode protégé.
  PS           Bascule vers la gestion particulière A20 nécessaire sur
               les machines PS/2.
  GEODE        Supprime les délais lors du changement de la ligne A20.
               Il a été signalé que cela aide au fonctionnement de FDXMS
               sur le processeur Geode GX1 de National Semiconductor.
  DELAY=       Le nombre d'appels à la routine de délai après avoir
               chargé la ligne A20. Si vous recevez le message d'erreur
               "Unable to switch A20 address line" c'est-à-dire
               "Impossible de changer la ligne d'adresse A20", essayez
               de le régler sur une valeur élevée (disons 4000) et voyez
               si cela aide. Puis faites le réglage fin pour une valeur
               aussi faible que possible pour la vitesse. La valeur par
               défaut est 1. La valeur maximale est 65535 et 0
               correspond à 65536.

Commentaires :

  FDXMS.SYS doit être chargé dans CONFIG.SYS / FDCONFIG.SYS.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS :
    device=c:\fdos\fdxms.sys
