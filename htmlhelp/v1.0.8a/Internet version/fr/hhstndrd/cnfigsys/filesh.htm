Commande : fileshigh

  FILESHIGH indique à FreeDOS combien de fichiers il devrait pouvoir
  ouvrir simultanément.
  FILESHIGH est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  fileshigh=n
   où n est un nombre de fichiers dans la plage 8-255. La valeur par
   défaut est 8, mais un nombre correct pour la plupart des systèmes
   modernes est 20 - 40. Certains programmes peuvent avoir besoin d'un
   réglage plus élevé pour fonctionner correctement ; de tels programmes
   indiqueront généralement la valeur nécessaire dans leur documentation.

Options:

  - aucune -

Commentaires :

  FILESHIGH spécifie le nombre de fichiers qui seront autorisés à être
  ouverts en même temps (réserve la mémoire nécessaire pour prendre en
  charge l'ouverture de ce nombre de fichiers).
  Remarque : il existe d'autres restrictions, donc un programme donné
  peut ne pas être capable d'ouvrir ce nombre de fichiers. Un nombre
  correct est 20,  même si certains programmes suggèrent/nécessitent 30,
  40, ou même 255.
  FILESHIGH est similaire à FILES, sauf qu'il tente de se charger
  d'abord dans la mémoire haute (si cela ne marche pas, il devrait
  se charger dans la mémoire conventionnelle).
  Remarque : l'ordre avec lequel vous chargez les périphériques peut
  avoir un grand impact sur la quantité de mémoire libre disponible.
  En général essayez de charger les gros (en termes d'utilisation
  mémoire) programmes d'abord dans la mémoire haute.
  Important : vous devriez avoir un gestionnaire de mémoire haute tel
  que (HIMEM) ou HIMEMX ou bien JEMMEX d'installé (device=himem.exe
  ou device=himemx.exe avant dans CONFIG.SYS / FDCONFIG.SYS) avant
  d'utiliser cette option.
  FILESHIGH est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    fileshigh=20
