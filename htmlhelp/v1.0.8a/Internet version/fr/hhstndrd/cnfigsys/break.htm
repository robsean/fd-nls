Commande : break

  BREAK active (valeur par défaut) ou désactive la vérification étendue
  de CTRL+C / Control-Break. Lorsqu'elle est activée, le kernel
  effectuera la vérification (et invoquera le gestionnaire actuel si
  présent) avant la plupart des appels int 21h. Lorsqu'elle est
  désactivée, le kernel n'effectue la vérification sur les appels E/S
  qu'en utilisant les flux standards.
  BREAK peut être utilisée dans CONFIG.SYS / FDCONFIG.SYS et
  lorsque FreeDOS est EN COURS DE FONCTIONNEMENT.

Syntaxe :

  BREAK=[ON | OFF]

Options :

  ON   CTRL+C est également testé lors d'un accès à une
       disquette ou au disque dur.
  OFF  CTRL+C est testé seulement s'il y a une sortie sur
       l'écran. Valeur par défaut = OFF

Commentaires :

  Tapez "BREAK" sans paramètres pour afficher le réglage actuel de
  BREAK. Si BREAK est sur ON, les programmes et les commandes peuvent
  toujours être annulés en tapant CTRL+C.
  Si BREAK est sur OFF, les programmes et les commandes ne peuvent
  être annulés que si vous avez une sortie sur l'écran.
  BREAK peut être utilisée dans CONFIG.SYS / FDCONFIG.SYS mais elle
  est aussi une commande COMMAND.COM.
  BREAK est une commande interne à kernel.sys et ne nécessite aucun
  autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
   break=on   les programmes peuvent être annulés avec CTRL+C
   break=off  les programmes ne peuvent être annulés avec CTRL+C
