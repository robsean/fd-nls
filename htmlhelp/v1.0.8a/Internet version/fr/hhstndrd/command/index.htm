Commande : command (command.com ou FreeCom)

  COMMAND démarre une nouvelle copie du shell de commandes de FreeDOS.

Syntaxe :

  COMMAND [[lecteur]chemin] [périphérique] [/E:nnnnn] [/L:nnnn] [/U:nnn]
          [/P] [/MSG] [/LOW] [/Y [/[C|K] commande]]
  COMMAND [/?]
          [lecteur]chemin  Spécifie le dossier contenant COMMAND.COM,
                       par exemple C:\exemple\
          lecteur      La lettre du lecteur, par exemple C:
          chemin       Le répertoire, par exemple \exemple\
          périphérique Spécifie le périphérique à utiliser pour l'entrée
                       et la sortie des commandes.

Options :

  /E:nnnnn    Règle la taille initiale de l'environnement sur nnnnn
              octets. (nnnnn devrait être entre 256 et 32,768).
  /L:nnnn     Spécifie la taille des tampons internes (nécessite
              également /P). (nnnn devrait être entre 128 et 1,024).
  /U:nnn      Spécifie la taille du tampon d'entrée (nécessite
              également /P). (nnn devrait être entre 128 et 255).
  /P          Rend le nouveau shell de commandes permanent (impossible
              de quitter).
  /MSG        Stocke tous les messages d'erreur en mémoire (nécessite
              également /P).
  /LOW        Force le shell de commandes à garder ses données
              résidentes dans la mémoire basse.
  /Y          Parcourt le programme batch spécifié par /C o
  /C commande Exécute la commande spécifiée et retourne
  /K commande Exécute la commande spécifiée et continue
  /?          Affiche l'aide

Commentaires :

  [lecteur]chemin  Le lecteur et le chemin où le shell doit rechercher
               la partie transitoire tu du programme. C'est généralement
               nécessaire seulement pour définir le COMSPEC.
  périphérique Périphérique alternatif pour toutes les e/s.
  /e:nnnnn     La taille de l'environnement, en octets, dans la plage
               160-32768.
               Ce nombre sera arrondi aux 16 octets les plus proches.
               La valeur par défaut est 256.
  /p           Rend le shell permanent, donc la commande EXIT ne quitte
               pas le shell.
  /c {chaîne}  Exécute la commande dans {chaîne}, puis quitte.
  /msg         Charge dans la mémoire tout message d'erreur qui pourrait
               être stocké sur le disque.
  Command (command.com) est aussi appelé FreeCom. Il contient de
  nombreuses commandes internes qui ne nécessite aucun autre fichier
  pour fonctionner.
  Vous pouvez retrouver la liste des commandes internes ici.

Exemples :

  C:\command /E:1024 /L:1024 /U:128 /P
  command /?
