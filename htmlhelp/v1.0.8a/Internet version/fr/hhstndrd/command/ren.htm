Commande : ren

  REN renomme un fichier/répertoire ou des fichiers/répertoires.

Syntax:

  REN [lecteur][chemin][nomdossier1 | nomfichier1]
         [nomdossier2 | nomfichier2]
         lecteur           La lettre du lecteur, par exemple : C:
         chemin            Le répertoire, par exemple : \exemple\ ,
                           en entier : "C:\exemple\"
         nomdossier1       Le nom de l'ancien sous-répertoire, par
                           exemple :
                           \dir_old , en entier : "C:\exemple\dir_old"
         nomdossier2       Le nom du nouveau sous-répertorie, par
                           exemple: dir_new , en entier : "dir_new"
         nomfichier1       L'ancien nom de fichier, par exemple :
                           \old_file.txt
                           en entier : "C:\exemple\old_file.txt"
         nomfichier2       Le nouveau nom de fichier, par exemple:
                           new_file.txt
                           en entier : "new_file.txt"

Options :

  - aucune -

Commentaires :

  Vous pouvez également utiliser RENAME au lieu de REN.
  Notez que vous ne pouvez pas spécifier un nouveau lecteur ou chemin
  pour la destination. Pour cela, utilisez la commande MOVE.
  Vous pouvez utiliser les jokers (* et ?) pour spécifier un jeu de
  fichiers. Si vous utilisez des jokers dans nomfichier1, vous devez
  utiliser des jokers à la même position dans nomfichier2.
  Sous DOS pur, les noms de répertoires ne doivent pas dépasser 8
  lettres. Les noms de fichier utilisant 8.3 signifie : 8 lettres pour
  le nom, 3 lettres pour l'extension, par exemple exemple1.txt.
  REN est une commande interne à command.com et ne nécessite aucun
  autre fichier pour fonctionner.

Exemples :

  ren C:\exemple\dir_old dir_new
  ren C:\exemple\old_file.txt new_file.txt

Voir également :

  move
  rename
