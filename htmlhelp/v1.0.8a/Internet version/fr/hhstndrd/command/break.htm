Commande : break

  BREAK active (valeur par défaut) ou désactive la vérification étendue
  de CTRL+C / Control-Break. Lorsqu'elle est activée, le kernel
  effectuera la vérification (et invoquera le gestionnaire actuel si
  enfoncée) avant la plupart des appels int 21h. Lorsqu'elle est
  désactivée, le kernel effectue la vérification lors des appels E/S
  en utilisant les flux standards.
  BREAK peut être utilisé dans CONFIG.SYS / FDCONFIG.SYS (cf. aussi ici)
  et lors du fonctionnement de FreeDOS.

Syntaxe :

  BREAK [ON | OFF] [/?]

Options :

  ON   CTRL+C          est même testé lors d'un accès à une disquette
                       ou DD
  OFF  CTRL+C          est seulement testé s'il y a une sortie sur
                       l'écran. La valeur par défaut est OFF
  /?   Affiche l'aide

Commentaires :

  Tapez "BREAK" sans paramètre pour afficher le réglage actuel de BREAK.
  Si BREAK est ACTIVÉ, les programmes et les commandes peuvent toujours
  être interrompus en appuyant sur CTRL+C.
  Si BREAK est sur OFF, les programmes et les commandes ne peuvent être
  interrompus que s'il y a une sortie sur l'écran.
  BREAK peut être utilisé dans CONFIG.SYS / FDCONFIG.SYS (cf. aussi ici)
  et lors du fonctionnement de FreeDOS.
  BREAK est une commande interne à kernel.sys et ne nécessite aucun
  autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS / SUR LA LIGNE DE COMMANDE :
    break on   les programmes peuvent être interrompus par CTRL+C
    break off  les programmes ne peuvent pas être interrompus par
               CTRL+C
