Befehl: menudefault

  MENUDEFAULT setzt eine zeitverzögerte Standardoption für ein 
  config.sys / fdconfig.sys Auswahlmenue.
  MENUDEFAULT ist ein CONFIG.SYS / FDCONFIG.SYS-Befehl.

Syntax:

    menudefault=Standardoption,Verzögerung

Optionen:

  Verzögerung    Verzögerungszeit in Sekunden.
  Standardoption  Verwendete Standardoption, nachdem die Verzögerungs-
                  zeit vorbei ist.

Kommentar:

  MENUDEFAULT setzt eine zeitverzögerte Standardoption für ein 
  config.sys / fdconfig.sys Auswahlmenue: Wenn während der 'Verzöge-
  rungs"-Sekunden keine Taste gedrückt wurde, wird die Standardauswahl
  aktiviert.
  Das Drücken einer Taste stoppt den Countdown, der Kernel wird endlos
  warten, bis der Benutzer eine Auswahl getroffen hat. Bitte beachten Sie,
  daß einige virtuelle Maschinen wie Bochs ein defektes (zu schnelles)
  Timing haben können.
  MENUDEFAULT ist ein interner Befehl, d.h. der Befehl ist in der
  kernel.sys enthalten. Es wird zur Ausführung des Befehls kein 
  zusätzlicher Befehl (Datei) benötigt.

Beispiel:

  IN DER CONFIG.SYS / FDCONFIG.SYS:
    menucolor 15,1
    MENU Bitte wählen Sie:
    MENU
    MENU 0)    normale Treiber (himem und emm386)
    MENU 1)    alternative Treiber (himemx und jemm386)
    MENUDEFAULT=0,5
    0?DEVICE=A:\himem.exe
    1?DEVICE=A:\himemx.exe
    0?DEVICE=A:\emm386.exe
    1?DEVICE=A:\jemm386.exe
  Das Menue wartet 5 Sekunden. Wenn in dieser Zeit keine Taste (Zahl)
  gedrückt wurde, wird automatisch 0 gewählt. menucolor 15,1 ändert
  die Farben auf weißen Text auf blauem Hintergrund.
    0 wählt himem.exe und emm386.exe
    1 wählt himem.exe und jemm386.exe
