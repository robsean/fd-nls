Befehl: install

  INSTALL startet ein TSR-Programm (ein Programm, das im Speicher bleibt)
  ohne eine Umgebung, anstelle von Befehlen wie lh oder loadhigh.
  INSTALL lädt ein TSR-Programm in den unteren oder konventionellen
  Speicher, während INSTALLHIGH das TSR-Programm in den oberen Speicher
  lädt.
  INSTALL ist ein CONFIG.SYS / FDCONFIG.SYS-Befehl.

Syntax:

  install=[[Laufwerk][Pfad][Dateiname]] [Optionen]
          Laufwerk   Der Laufwerksbuchstabe, z.B.: C:
          Pfad       Der zu wählende Pfad, z.B.: \Beispiel\
          Dateiname  Die zu startende TSR-Datei, z.B.: test.sys

Optionen:

  Optionen  Das sind die Optionen des zu startenden TSR-Programms. Es
            handelt sich nicht um Optionen des 'install'-Befehls.

Kommentar:

  INSTALL lädt das durch Laufwerk:\Pfad\Dateiname spezifizierte Programm.
  Es wird normalerweise dazu benutzt, TSR Programme (TSR = beende und
  bleibe resident im Speicher) mit einem minimalen Umgebungsblock zu
  laden.
  Die Optionen sind für den Treiber selbst; vgl. auch die Dokumentation
  zu den Optionen und ihre Verwendung, die mit dem speziellen Treiber
  geliefert wurde.
  INSTALL ist ein interner Befehl, d.h. der Befehl ist in der kernel.sys
  enthalten. Es wird zur Ausführung des Befehls kein zusätzlicher
  Befehl (Datei) benötigt.

Beispiel:

  IN DER CONFIG.SYS / FDCONFIG.SYS:
    install=C:\fdos\bin\ctmouse.exe
