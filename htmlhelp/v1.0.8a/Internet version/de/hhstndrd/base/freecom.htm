Befehl: FreeCom (command.com)

  FREECOM (command.com) startet eine neue Kopie der 
  FreeDOS Command Shell.

Syntax:

  COMMAND [[Laufwerk]Pfad] [Gerät] [/E:nnnnn] [/L:nnnn] [/U:nnn] [/P] 
          [/MSG] [/LOW] [/Y [/[C|K] Befehl]]
  COMMAND [/?]
          [Laufwerk]Pfad   Spezifiziert das Verzeichnis, das die 
                           COMMAND.COM entält, z.B. C:\Beispiel\
          Laufwerk         Der Laufwerksbuchstabe, z.B.: C:
          Pfad             Der zu wählende Pfad, z.B.: \Beispiel\
          Gerät           Spezifiziert das für die Befehlsein- und 
                           -ausgabe zu benutzende Gerät.

Optionen:

  /E:nnnnn   Setzt die anfängliche Umgebungsgröße auf nnnnn Bytes.
             (nnnnn sollte eine Zahl zwischen 256 und 32.768 sein).
  /L:nnnn    Spezifiziert die Größe der internen Puffer (erfordert 
             ebenfalls /P). (nnnn sollte eine Zahl zwischen 128 und 1.024
             sein).
  /U:nnn     Spezifiziert die Eingabepuffergröße (erfordert auch /P).
             (nnn sollte eine Zahl zwischen 128 und 255 sein).
  /P         Richtet die neue Command Shell Instanz dauerhaft ein (kann 
             nicht beendet werden).
  /MSG       Speichert alle Fehlermeldungen im Speicher (erfordert 
             auch /P).
  /LOW       Zwingt die Command Shell, bleibende Daten im LowMem zu 
             speichern.
  /Y         Arbeitet schrittweise die durch /C oder durch /K 
             spezifizierten Stapelverarbeitungsdaten ab.
  /C Befehl  Führt den spezifizierten Befehl aus und kehrt zurück.
  /K Befehl  Führt den spezifizierten Befehl aus und läuft weiter.
  /?         Zeigt die Hilfe.

Kommentar:

  [Laufwerk]Pfad  Das Laufwerk und der Pfad, an dem die Shell nach dem 
                  vorübergehend anzuwendenden Teil des Programms schauen
                  soll. Dies wird normalerweise nur benötigt, um COMSPEC
                  zu setzen.
  Gerät          Ein alternatives Gerät für alle I/O
  /e:nnnnn        Die Umgebungsgröße in Bytes im Bereich von 160 - 
                  32.768 Bytes.
                  Die Zahl wird auf die nächsten 16 Bytes gerundet. 
                  Standardwert ist 256 Byte.
  /p              Macht die Shell permanent, so daß der EXIT-Befehl die
                  Shell nicht beenden kann.
  /c {string}     Führt den in {string} angegebenen Befehl aus, sobald 
                  command geladen ist. Nach der Verarbeitung des 
                  angegebenen Befehls wird command wieder beendet.
  /msg            Lädt jede Fehlermeldung, die auf Diskette gespeichert
                  werden könnten in den Arbeitsspeicher. 
  Der Befehl command (command.com) wird auch FreeCom genannt. Er entält
  eine Menge von internen Befehlen, die keine eigene Datei benötigen, um
  ausgeführt zu werden. Sie finden die Liste der internen Befehle hier.

Beispiel:

  C:\command /E:1024 /L:1024 /U:128 /P
  command /?
