Netzwerken unter FreeDOS - MS Client

Wo man sie findet

    (Bild des MS Client 3.0)
  Microsofts Programme "LAN manager" und "MS Client 3.0" können von
  Microsofts ftp-Seite heruntergeladen werden und sind frei zur "internen
  Verwendung", aber nicht zum Vertrieb.
    * MS LAN manager Client (1998) finden Sie unter:
        ftp://ftp.microsoft.com/bussys/Clients/LANMAN/ (*01)
    * MS Client 3.0 (1995) finden Sie unter:
        ftp://ftp.microsoft.com/bussys/Clients/MSCLIENT/ (*02)

MS Client als Server

  Es ist möglich (wird aber nicht offiziell unterstützt), ein Microsoft
  Update dazu zu verwenden, bei MS Client Serverfunktionalität hinzuzu-
  fügen. Dieses Update heißt WG1049.EXE und kann hier heruntergeladen
  werden:
    ftp://ftp.microsoft.com/softlib/mslfiles/wg1049.exe (*16)
  Im Ergebnis fügt dieses Tool Features hinzu, die mit denen von
  "Workgroup Add-on für DOS" vergleichbar sind, welches von Microsoft
  nicht mehr länger verkauft wird.
    (Bild DOS Workgroup Add-on für MS-DOS)

Kritik

  Die Integrationsmöglichkeiten einer DOS Maschine in ein modernes Win-
  dows XP oder eine GNU/Linux SMB Arbeitsgruppe sind dennoch begrenzt.
  Zum Beispiel scheint es nicht möglich zu sein, mit GNU/Linux und
  smbmount auf einen DOS Share zugreifen zu können. Und die Verwendung
  des TCP/IP als Protokoll unter MS Client verbraucht sehr viel Ar-
  beitsspeicher und kann zu einem instabilen DOS System führen.
  Steven Baker bemerkt kritisch (vgl.: http://web.archive.org/web/
  20070314043508/http://www.ece.ubc.ca/~gillies/9802net.html (*17)), daß,
  während das grundlegende TCP/IP Protokoll über die Jahre stabil blieb,
  Microsofts SMB Protokolle sich von Upgrade zu Upgrade veränderten und
  "in einem Dutzend verschiedener Dialekte" bestehen. Deshalb ist Kompa-
  tibilität ein Thema. Andere Probleme können durch die Authentifizie-
  rungsmethoden verursacht werden, die von Windows oder Samba verwendet
  werden.
  Abgesehen davon kann man es als gute Sache sehen, daß Microsoft es
  immer noch zuläßt, daß man diese Software legal herunterladen und
  verwenden darf. Einige Dateien vom "MS Client" Paket sind immer noch
  wichtig für die Verwendung von TCP/IP Applikationen über die heute
  üblichen NDIS Treiber.

Alternativen

  * NFS
  * Sie könnten einen FTP Server auf DOS verwenden, der im Hintergrund
    läuft (als Datalight Sockel, was hier erklärt wird (vgl.: http://
    sourceforge.net/mailarchive/message.php?msg_name=48BC5752.3090303%40
    mainz-online.de (*18)) und ein Tool um einen FTP Server auf einen
    Laufwerksbuchstaben zu mappen (für Windows z.B. netdrive
    siehe: http://www.netdrive.net/ (*19)) um Ihre DOS Maschine in ein
    Netzwerk zu integrieren. GNU/Linux kann sogar FTP Server ohne ein
    Extratool mounten.
