Netzwerken unter FreeDOS - Andere TCP/IP Kernel

  Einige andere externe TCP/IP Kernels können auch legal heruntergeladen
  und verwendet werden.
    * Novell verteilt seinen beliebten 16-Bit TCP/IP Protokollstack
      "TCP16.EXE" (vgl.: http://bbs.actapricot.org/files/area37/tcp16.exe
      (*01)), der direkt oberhalb des ODI Treibers läuft. Einige Programme
      wie Josh's "Tiny" remote control software (vgl.: http://josh.com/
      tiny/ (*02)) und Tsoft's "NFS Clients for DOS" (vgl.: http://www.
      rawbandwidth.com/software/ (*03)) verwenden diesen Kernel immer noch.
    * Microsoft's "MS Client 3.0" (vgl.: ftp://ftp.microsoft.com/bussys/
      Clients/MSCLIENT/ (*04)) entält seinen eigenen TCP/IP Kernel
      "TCPTSR.EXE". Er kann auch von externen Programmen verwendet werden.
      Es erscheint aber sinnvoll, zuerst die Arbeitsspeichervoraus-
      setzungen zu prüfen. TCP/IP von MS Client scheint eine Menge
      regulären Arbeitsspeicher zu verwenden.
  Für weitergehende Informationen besuchen Sie bitte die Webseiten von
  Microsoft und Novell.
