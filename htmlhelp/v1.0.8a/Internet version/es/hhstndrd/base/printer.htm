Comando: printer.sys

  PRINTER.SYS proporciona capacidades de administración de página de
  códigos para un controlador de impresora. PRINTER.SYS se debe cargar
  en CONFIG.SYS / FDCONFIG.SYS. Cuando FreeDOS ya se está ejecutando,
  puede cargar PRINTER.SYS más tarde con DEVLOAD.

Sintaxis:

  printer [Disp_nomb][:]=([TipoHardware],[{n|(n1,[n2,]...)}],[m])

Opciones:

  Disp_nomb     Nombre del dispositivo del puerto de impresora que se va
                a expandir, como PRN o LPTx. El color es opcional.
  TipoHardware  El identificador del tipo de adaptador de pantalla de
                hardware. También puede determinar la cantidad de
                subfondos a usar. Todavía no hay tipos compatibles.
  n, n1, n2,..  Números para los números de página de códigos de las
                páginas de códigos de hardware.
  m             Número de buffers que desea especificar para preparar
                páginas de códigos (el valor predeterminado es 1)

Comentarios:

  Este controlador aún no tiene una forma binaria, porque todavía no
  se ha agregado una administración específica del hardware de la
  impresora (puede compilar tanto PRINTER.SYS como DISPLAY.SYS desde las
  fuentes de DISPLAY).

Ejemplos:

  Cuando PRINTER.SYS tenga algún soporte de hardware, podrá hacer:
    PRINTER PRN=(IBMPro,437,1)
  Usted especifica que tiene una impresora compatible con IBM Proprinter,
  con la página de códigos de hardware 437, y solicita solo 1 búfer, para
  minimizar la huella de memoria residente.
