Comando: display

  DISPLAY carga las capacidades de administración de la página de
  códigos de la consola.

Sintaxis:

DISPLAY nombre_dispositivo[:]=(Tipo_hardware,[n|(n1[,n2]...)],[m|(m,p)])
        [/V] [/C] [/NOHI]

Opciones:

  nombre_dispositivo  Un nombre de dispositivo válido (que se ignora en
                      esta versión).
                      Use cualquiera (como CON).
  Tipo_herdware       Una etiqueta de tipo de hardware compatible (como
                      EGA o VGA)
  n                   Página de códigos de hardware predeterminada (para
                      una página de códigos de hardware).
                      Normalmente será 437.
  n1, n2, ...         Páginas de códigos de hardware, si hay más de una
                      (generalmente tendrá para usar controladores 
                      ARABICOS/HEBREOS de terceros).
  m                   Número de búfer para páginas de códigos preparadas
                      (1 por defecto).
  p                   Parámetros específico de hardware adicional.
  /C                  Exclusivo: no se carga si DISPLAY ya está presente.
  /NOHI               Desactiva el uso de la memoria XMS para las páginas
                      de códigos preparadas.
  /V                  Verboso: información extendida.

Comentarios:

Hardware soportado:
  VGA     VGA adaptadores. Se utilizarán 3 subfondos en cualquier caso
          adaptadores EGA.
  EGA     Determine automáticamente el número de subfondos que se
          utilizarán.
  EGA 14  Obligar a usar dos fuentes secundarias (8x8 y 8x14).
  EGA 8   Obligar a usar solo una subfondos (8x8).
  LCD     Adaptadores LCD (igual que EGA 8).
  
          Para todos los adaptadores anteriores, el parámetro p intentará
          forzar el número de subfondos a utilizar (1=8x8, 3=8x8, 8x14,
          3=8x8, 8x14, 8x16) siempre que el hardware lo permita.
  CGA     Adaptador CGA (FreeDOS GRAFTABL). El parámetro p se ignora.
  DISPLAY no produce EXITCODES, pero proporciona un conjunto de arrores a
  través de la función int2Fh/AD03h para cada función Debe verificarlo
  después de cada operación fallida.

  Código de Error   Operación     Significado
  -------------------------------------------------------------------
  1                 Todas         Función desconocida
  26                Seleccionar   La página de códigos no estaba
                                  preparada
  26                Consulta      No se seleccionó ninguna página
                                  de códigos
  27                Seleccionar   KEYB no pudo cambiar la página
                                  de códigos
  27                Consulta      Error del dispositivo
  27                Prepararse    La página de códigos no se encontró
                                  en el archivo
  29                Seleccionar   Error del dispositivo
  29                Prepararse    Error del dispositivo OR XMS error
  31                Prepararse    Archivo fuente dañado OR XMS error
  31                Fin preparar  No empieza a preparar para buscar
  -------------------------------------------------------------------

Ejemplos:

  DISPLAY CON=(VGA,437,1)
    VGA adaptador, cuya página de códigos estándar de BIOS es 437. Se
    utiliza 1 búfer para la página de códigos de software. Este es el
    estándar para la mayoría de los usuarios.
                 
  DISPLAY CON=(EGA,,2)
    El adaptador de gráficos es EGA o mejor, pero se desconoce. La
    página de códigos predeterminada del BIOS también es desconocida.
    Se requieren dos amortiguadores.
