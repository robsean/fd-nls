Comando: jemmex

  JEMMEX es un administrador de memoria expandido que ya tiene un XMM
  incluido. JEMMEX tiene que cargarse en CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  DEVICE=[unidad][ruta][jemmex.exe] [opciones]
         unidad  La letra de unidad, p. ej. C:
         ruta    El directorio, p. ej. \ejemplo\
  o ejecútelo desde la línea de comando:
  [unidad][ruta][jemmex.exe] [opciones]

Opciones:

 +A20/NOA20     A20-desactiva la emulación activado/desactivado (opción
                predeterminada activada).
  A20METHOD:m   Establecer el método de cambio A20. Los valores posibles
                para m son KBC, PS2, BIOS, FAST, PORT92 y ALWAYSON.
  ALTBOOT       Use una estrategia de reinicio alternativa.
  B=####        Especifique la dirección del segmento más bajo para la
                banca EMS (predeterminado=4000).
  D=###         Establece el tamaño del búfer DMA en kB 
                (predeterminado=64, el máximo es 128).
  EMX           Mayor compatibilidad con el extensor EMX DOS.
  FASTBOOT      Reinicio rápido. Requiere que JemmEx se cargue en 
                CONFIG.SYS.
  FRAME=E000    Establecer marco de página EMS (FRAME=NONE desactiva el
                marco). Se hacepta algún valor entre 8000 y E000, pero no
                todos funcionarán.
  HMAMIN=n      Establecer la cantidad mínima en Kb para arignar el HMA.
  I=D000-D7FF   Forzar el uso de una región para UMB. Sin esta opción, el
                rango C000-EFFF se escanea en busca de páginas no
                utilizadas. También se puede usar para agregar (partes 
                de) las regiones A000-BFFF o F000-F7FF como UMB. ¡No use
                esta opción si no sabe lo que está haciendo!
  I=TEST        Escanea páginas de la ROMs no utilizadas, incluye 
                regiones encontradas como UMBs.
  [MAX=]#####   Límite para memoria VCPI (y EMS si <32M) en kB
                (predeterminado 120 MB).
  MAXEXT=n      Limite el uso de memoria extendida en n kB.
  MIN=#####     Reserve hasta ##### kB para la memoria EMS / VCPI en init
                (predeterminado=0).
  NOCHECK       No permite el acceso al espacion de direcciones sin RAM
                (MMIO).
  NOE801        No use int 15h, E801h para obtener la cantidad de memoria
                extendida.
  NOE820        No use int 15h, E820h para obtener la cantidad de memoria
                extendida.
  NOEMS         Desactiva el manejo de EMS.
  NODYN         Sin asignación dinámica de memoria XMS (use MIN= para
                establecer la cantidad fija).
  NOHI          No mueva la parte residente a la primera UMB.
  NOINVLPG      No utilice el código de operación INVLPG.
 +PGE/NOPGE     Activación/Desactivación del uso de la función de
                habilitación global de la página (opción predeterminada
                desactivada).
  RAM/NORAM     Intenta suministrar UMBs activado/desactivado
                (predeterminado activado).
  S=D000-D7FF   Asumir la activación de la Shadow-RAM por UMBPCI,
                inclúyalo como UMB
  SB            Modo de compatibilidad del controlador SoundBlaster
  SPLIT         Recupera páginas EPROM 4k parcialmente usadas para UMBs.
 +VCPI/NOVCPI   VCPI soporte activado/desactivado (predeterminado 
                activado).
  VDS/NOVDS     Servicios virtales DMA activado / desactivado
                (predeterminado activado).
 +VME/NOVME     Extensiones V86-Modo activado / desactivado
                (predeterminado activado).
  VERBOSE       Muestra detalles adicionales durante el inicio
                (abreviatura: /V).
  X=D000-D7FF   Excluir la región de ser tocada o utilizada por JemmEx.
  X=TEST        Escanee la región de memoria C000-EFFF para la exlusión
                UMS.
  X2MAX=n       Máximo valir en kB para el informe de memoria libre de 
                XMS V2. (predeterminado=65535).
  XMSHANDLES=n  Numero de manejadores XMS disponibles (8<=n<=128, 
                predeterminado=32).
  /?            Muestra la ayuda.
  '+': la opciónse puede establecer dinámicamente ejecutando JemmEx desde
       la línea de comandos.
  Cuando se invoca desde la línea de comandos, JemmEx comprenderá 
  adicionalmente:
  LOAD          instalar
  UNLOAD        desinstalar

Comentarios:

  JEMMEX requiere una CPU 80386+. Cargue JEMMEX como DEVICE=JEMMEX.EXE
  en CONFIG.SYS (¡aunque es un archivo ".exe"!).
  JEMMEX tiene que cargarse en CONFIG.SYS / FDCONFIG.SYS.
  Lo más probable es que JEMMEX sea la mejor opción que HIMEMX / JEMM386
  porque necesitará menos memoria de DOS que un XMM + JEMM386.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    device=c:\fdos\bin\jemmex.exe
