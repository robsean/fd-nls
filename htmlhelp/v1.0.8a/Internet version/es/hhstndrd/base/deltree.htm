Comando: deltree

  DELTREE elimina un árbol de directorios completo, incluidos los
  subdirectorios.

sintaxis:

  DELTREE [/Y] [/V] [/D] [/X] [/Z:EN_SERIO] arbol_dir [arbol_dir...]
  DELTREE [/?]
          unidad  La letra de unidad, por ejemplo C:
          ruta    El directorio, p. \ejemplo

Opciones:

  /Y            Elimina los directorios especificados sin pedir
                confirmación.
  /V            Informa el número de archivos y subdirectorios 
                eliminados, y su tamaño total.
  /D            Muestra la información de depuración.
  /X            para pruebas; en realidad no borres nada.
  /Z:SERIOUSLY  No preguntar cuando se especifica el directorio raíz.
  arbol_dir     Elárbol de directorios que se eliminará, p. C:\OLDTREE
                Tenga en cuenta que se pueden eliminar varios árboles
                de directorios con un solo comando.
  /?            Muestra la ayuda.

Comentarios:

  Por razones de rendimiento DELTREE sobrescribe la primera letra de cada
  nombre de archivo / nombre de directorio con un '?'. No elimina el
  archivo en sí, pero desasigna el espacio donde se escribe el contenido
  del archivo. FreeDOS, como otros DOS, reconoce el archivo / directorio
  renombrado (myfile.txt se convierte en ?yfile.txt; nom_ruta se 
  convierte en ?om_ruta) como eliminado, ya no lo muestra y no ofrece
  acceso a él. Los programas como DEBUG que tienen acceso de sector
  pueden leer el contenido del archivo / directorio. Mientras no escriba
  en la unidad, tiene la oportunidad de restaurar los archivos / 
  directorios nuevamente con UNDELETE, solo se perderá el primer carácter
  del nombre del archivo / directorio (_yfile.txt / _athname). Las únicas
  formas de eliminar el archivo CONTENIDO PARA CIERTOS ABSOLUTAMENTE es
  RENAME (REN) el archivo / directorio primero (por ejemplo, "a." / 
  "A") y eliminarlo más tarde (solo correcto en 8.3 - no con nombres 
  largos de archivo / nombres de directorios!)
  Los interruptores deben estar antes de arbol_dir en la línea de
  comandos: sintaxis compatible con MS. De lo contrario, serán ignorados.
  Las especificaciones de archivos pueden nombrar archivos, subdirectorios
  o listas de archivos de estilo DR DOS. ¡DELTREE es un comando peligroso!
  úselo bajo su propio riesgo. DELTREE tiene los siguientes EXITCODES 
  (ERRORLEVEL):
    00 Exito
    01 Algo no pudo ser eliminado
    03 Usuario abortar (Q,control-C)
    16 Error de sintaxis general o desbordamiento del búfer.
    17 No hay suficiente memoria
    18 Se requiere DOS 3.0 o superior
    19 Problema con el archivo de lista
    20 Desbordamiento interno del búfer
    21 Error al resolver el nombre del directorio (dots_fix)

Ejemplos:

  deltree /y c:\oldtree
  deltree c:\test2\   solicita cada subdirectorio de \test2 si realmente
                      desea eliminarlo
  deltree c:\test2    elimina todo el sobdirectorio de \test2
