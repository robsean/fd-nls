Redes de FreeDOS - PKT2ODI

  Como alternativa a la utilizada más comúnmente ODIPKT.COM algunas
  personas también les podría gustar probar Calderas PKT2ODI.EXE como
  controlador de paquetes al convertidor de ODI. Pero debemos ser
  conscientes de que la licencia para WebSpyder de Novell (que se aplica
  a PKT2ODI.EXE) sólo permite utilizar el software con el propósito de
  evaluar la misma. Para los usuarios comerciales del período de
  evaluación se limita a 90 días. Para los usuarios no comerciales se
  permite la evaluación de "un plazo razonable" de tiempo. (¿Qué tan
  razonable puede ser la evaluación de este software en el siglo 21?)
  De todos modos, PKT2ODI.EXE se iniciará en AUTOEXEC.BAT con
  un comando como este:
    LH C:\NETWORK\NWCLIENT\PKT2ODI.EXE /B:1 /I:69
  La opción "/B" indica a PKT2ODI.EXE que abordaje lógico debería
  utilizar. Como se explica más arriba tenemos que elegir el abordaje
  que utiliza el marco ETHERNET II. A diferencia de ODIPKT, PKT2ODI
  empieza a contar con "1", de modo que si al NET.CFG se parece a
  este ejemplo:
    --- NET.CFG ---
      Link Driver 3C574
        FRAME ETHERNET_II
        FRAME ETHERNET_802.2
        FRAME ETHERNET_802.3
        FRAME ETHERNET_SNAP
  tenemos que usar el parámetro
    PKT2ODI.EXE /B:1
  para definir el uso del primer abordaje (el uno con ETHERNET_II).
  Recuerde que tenemos que usar la tarjeta con Ethernet II, de lo
  contrario el convertidor no será capaz de comunicarse con el
  controlador y se quejan de no encontrar el MLID (Multiple Conductor
  Link Interface).
  El segundo parámetro, el "/I" establece la interrupción de software
  (vector) utilizado por el controlador. Ya nos enteramos de que esto
  debe ser 0x60, pero por desgracia esto no funcionará con PKT2ODI. El
  programa no acepta vectores de interrupción 0x60 al 0x68. Podemos
  utilizar vector 0x69 con el parámetro:
    PKT2ODI.EXE /I:69
  El uso de un vector de interrupción inusual como 0x69 puede ser un
  problema para algunas aplicaciones TCP/IP. Ellos deben estar
  configurados para utilizar este vector. Algunos programas, como el
  servidor web SIOUX, pueden no funcionar correctamente, si el vector
  no es 0x60.
  Eso es todo. Después de iniciar la LSL (Link Soporte Layer), el MLID
  (Multiple Link Interface Driver) y el convertidor de la interfaz del
  controlador de paquetes ahora debería estar en funcionamiento.
