Comando: loadhigh (lh)

  LOADHIGH carga un programa en el área de memoria superior. Como 
  LOADHIGH carga el programa en la memoria superio,queda más espacio
  libre en la memoria baja o convencional para otros programas.
  LOADHIGH es un comando de ARCHIVO POR LOTES / AUTOEXEC.BAT.
  También se puede usar en la línea de comandos.

Sintaxis:

  LOADHIGH [unidad][ruta]nombre_archivo [parámetros]
  LOADHIGH [/L:región1[,mintamaño1][;región2[,mintamaño2]...] [/S] [/?]]
           [unidad][ruta]nombre_archivo [parámetros]
           unidad          La letra de unidad, por ejemplo C:
           ruta            El directorio, \ejemplo\
           nombre_archivo  El nombre del programa para cargar en el área
                           de memoria superior, por ejemplo edit.exe

Opciones:

  /L:región1[,mintamaño1][;región2[,mintamaño2]]...
                Especifica la(s) region(es) de memoria en las que carga
                el programa. Región1 especifica el número de la primera
                región de la memoria; mintamaño1 especifica el tamaño
                mínimo, si lo hay, para región1. Región2 y mintamaño2
                especifican el número y el tamaño mínimo de la segunda
                región, si corresponde. Puede especificar tantas 
                regiones como desee.
  /S            Reduce un UMB a su tamaño mínimo mientras se carga el
                programa.
  /?            Muestra la ayuda.
  parámetros    Estas son las opciones para el programa que está cargando.

Comentarios:

  LOADHIGH también se puede usar fuera de un archivo por lotes. Para 
  cargar un controlador o un programa en la memoria superior, primero debe
  cargar un controlador como (HIMEM), HIMEMX, JEMMEX, (EMM386) 
  o JEMM386 en config.sys. También puede usar LH en lugar de LOADHIGH.
  LOADHIGH es un comando interno de command.com y no necesita otro archivo
  para funcionar.

Ejemplos:

  EN UN ARCHIVO .BAT / AUTOEXEC.BAT:
    loadhigh c:\fdos\bin\edit.exe
