Comando: call

  CALL llama a un programa por lotes para otro.
  CALL es un comando de ARCHIVO POR LOTES / AUTOEXEC.BAT.

Sintaxis:

  CALL [/S | /N] [/Y] [unidad][ruta]nombre_archivo [parámetros-batch]
       unidad          La letra de unidad, por ejemplo C:
       ruta            El directorio, por ejemplo \ejemplo\
       nombre_archivo  El archivo por lotes que se llamará

Opciones:

  parámetros-batch   Especifica cualquier información de línea de
                     comandos requerida por el programa por lotes, por
                     ejemplo test.bat
  /S                 Aplica el intercambio de FreeCOM
  /N                 Evita el intercambio de FreeCOM
  /Y                 Habilita el modo de rastreo durante la ejecución
                     del comando.

Comentarios:

  Cuando se completa el archivo por lotes llamado, se reanuda el
  procesamiento del archivo por lotes original. Los archivos por lotes se
  pueden ejecutar directamente desde otro archivo por lotes sin el comando
  de llamada, pero el procesamiento del archivo por lotes original no se
  reanudará. CALL es un comando interno de command.com y no necesita 
  ningún otro archivo para funcionar.

Ejemplos:

  EN UN ARCHIVO .BAT / AUTOEXEC.BAT:
    call c:\fdos\bin\defrag.exe
