Comando: command (command.com o FreeCom)

  COMMAND inicia una nueva copia del shell de comandos de FreeDOS.

Sintaxis:

  COMMAND [[unidad]ruta] [dispositivo] [/E:nnnnn] [/L:nnnn] [/U:nnn] [/P]
          [/MSG] [/LOW] [/Y [/[C|K] comando]]
  COMMAND [/?]
          [unidad]ruta Especifica el directorio que contiene COMMAND.COM,
                       por ejemplo, C:\ejemplo\
          unidad       La letra de la unidad, por ejemplo C:
          ruta         El directorio, por ejemplo \ejemplo\
          dispositivo  Especifica el dispositivo a usar para el comando
                       entrada y salida.

Opciones:

  /E:nnnnn    Establece el tamaño del entorno inicial en nnnnn bytes.
              (nnnnn debe estar entre 256 y 32,768).
  /L:nnnn     Especifica la longitud de los buffers internos (requiere
              /P también). (nnnn debe estar entre 128 y 1,024).
  /U:nnn      Especifica la longitud del búfer de entrada (requiere /P
              también). (nnn debe estar entre 128 y 255).
  /P          Hace que el nuevo command shell sea permanente (no se puede
              salir).
  /MSG        Almacena todos los mansajes de error en la memoria (requiere
              /P también).
  /LOW        Obliga al shell de comandos a mantener sus datos residentes
              en la memoria baja.
  /Y          Pasa por el programa por lotes especificado por /C o /K
  /C comando  Ejecuta el comando especificado y lo devuelve
  /K comando  Ejecuta el comando especificado y continúa
  /?          Muestra la ayuda

Comentarios:

  [unidad]ruta La unidad y la ruta donde está el shell para buscar la
               parte transitoria del programa. Por lo general, esto solo
               es necesario para configurar el COMSPEC.
  dispositivo  Un dispositivo alternativo para todas las entradas y 
               salidas.
  /e:nnnnn     El tamaño del entorno, en bytes, en el rango 160-32768.
               Este número se redondeará a los 16 bytes más cercanos.
               El valor por defecto es 256.
  /p           Hace que el shell sea permanente, por lo que el comando
               EXIT no sale del shell.
  /c {cadena}  Ejecuta el comando en {cadena}, luego sale.
  /msg         Carga cualquier mensaje de error que pueda estar almacenado
               en el disco de memoria.
  Command (command.com) también se llama FreeCom. Contiene muchos comandos
  internos que no necesitan ningún otro archivo para funcionar. Puedes
  encontrar la lista de comandos internos aquí.

Ejemplos:

  C:\command /E:1024 /L:1024 /U:128 /P
  command /?
