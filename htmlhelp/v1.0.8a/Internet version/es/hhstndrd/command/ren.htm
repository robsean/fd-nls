Comando: ren

  REN cambia el nombre de un archivo/directorio o archivos/directorios.

Sintaxis:

  REN [unidad][ruta][nombre_directorio_1 | nombre_archivo_1]
      [nombre_directorio_2 | nombre_archivo_2]
      unidad              La letra de la unidad, por ejemplo C:
      ruta                El directorio, por ejemplo, \ejemplo\, completo:
                          "C:\ejemplo\"
      nombre_directorio_1 El nombre del antiguo subdirectorio, por
                          ejemplo, \dir_antiguo, completo: 
                          "C:\ejemplo\dir_antiguo"
      nombre_directorio_2 El nombre del nuevo subdirectorio, por ejemplo,
                          \dir_nuevo, completo: "C:\ejemplo\dir_nuevo"
      nombre_archivo_1    El nombre del antiguo archivo, p. ej., 
                          antiguo.txt
                          completo: "c:\ejemplo\antiguo.txt"
      nombre_archivo_2    El nombre del nuevo archivo, p. ej., nuevo.txt
                          completo: "c:\ejemplo\nuevo.txt"

Opciones:

  - ninguno -

Comentarios:

  También puede utilizar RENAME en lugar de REN.
  Tenga en cuenta que no puede especificar una nueva unidad o ruta de
  acceso para su destino.
  Utilice el comando MOVE para ese propósito.
  Puede utilizar comodines (* y ?) para especificar un conjunto de 
  archivos. Si utiliza comodines en nombre_archivo_1, debe utilizar
  comodines en la misma posición en nombre_archivo_2.
  En DOS puro el nombre del directorio no debe ser más largo que 8 letras.
  Los nombre de ficheros utilizan 8.3 significa: 8 letras para el nombre,
  3 letras para el final, por ejemplo, ejemplo1.txt
  REN es un comando interno a command.com y no necesita ningún otro
  archivo para poder trabajar.

Ejemplos:

  ren C:\ejemplo\dir_antiguo dir_nuevo
  ren C:\ejemplo\antiguo.txt nuevo.txt
