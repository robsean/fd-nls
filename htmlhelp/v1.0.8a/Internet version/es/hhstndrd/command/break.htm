Comando: break

  BREAK es un comando extendido de CTRL+C (STRG+C) / Control-rotura 
  comprueba si esta on [por defecto] o off. Cuando esta activado on, el
  núcleo realiza la comprobación (e invocará al controlador actual si 
  se presiona) antes de la mayoría de las llamadas a int 21h. Cuando está
  sesactivado, el núcleo sólo realizará la comprobación de las llamadas de
  I/O mediante secuencias estándar.
  BREAK se puede utilizar en CONFIG.SYS / FDCONFIG.SYS (véase también 
  aquí) y mientras se ejecuta FreeDOS.

Sintaxis:

  BREAK [ON | OFF] [/?]

Opciones:

  ON   CTRL+C (STRG+C) incluso es comprobado cuando existen accesos a 
       disquetes o Discos duros.
  OFF  CTRL+C (STRG+C) sólo se comprueba si hay una salida en pantalla.
       Por defecto es igual a OFF
  /?   Muestra la ayuda.

Comentarios:

  Escriba "BREAK" sin un parámetro para mostrar el ajuste de interrupción
  actual. Si el BREAK esta en ON, los programas y comandos siempre pueden
  abortarse escribiendo CTRL+C (STRG+C).
  Si BREAK esta en OFF, los programas y los comandos sólo pueden abortarse
  si se obtiene una salida en la pantalla.
  BREAK se puede utilizar en CONFIG.SYS / FDCONFIG.SYS (véase también 
  aquí) y mientras se ejecuta FreeDOS.
  BREAK es un comando interno de kernel.sys y no necesita ningún otro
  archivo para poder trabajar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS / LíNEA DE COMANDOS:
    break on   programas pueden ser abortados con CTRL+C (STRG+C)
    break off  programas no se pueden abortar con CTRL+C (STRG+C)
