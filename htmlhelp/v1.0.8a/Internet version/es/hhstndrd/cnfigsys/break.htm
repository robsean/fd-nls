Comando: break

  BREAK establece la comprobación extendida CTRL+C(STRG+C) / Control-Break
  en activado [predeterminado] o desactivado.
  Cuando se establece en ON, el nucleo realizará la comprobación (e 
  invocará el control actual si se presiona) antes de la mayoría de las 
  llamadas int 21h.
  Cuando se establece en OFF, el núcleo solo realiza la comprobación de
  las llamadas de E/S mediante secuencias estándar.
  BREAK se puede utilizar en CONFIG.SYS / FDCONFIG.SYS y mientras ejecutas
  FreeDOS.

Sintaxis:

  BREAK=[ON | OFF]

Opciones:

  ON   CTRL+C (STRG+C) también se prueba mientras se tiene acceso a
       disquete o disco duro (HD).
  OFF  CTRL+C (STRG+C) solo se prueba si hay una salida en la pantalla.
       Predeterminado = OFF

Comentarios:

  Escriba "BREAK" sin un parámetro para mostrar el ajuste BREAK actual.
  Si BREAK está activado, los programas y comandos siempre se pueden
  anular escribiendo CTRL+C (STRG+C).
  Si BREAK está desactivado, los programas y comandos solo se pueden
  anular si se obtiene una salida en la pantalla.
  BREAK se puede utilizar en CONFIG.SYS / FDCONFIG.SYS, pero también es
  un comando COMMAND.COM.
  BREAK es un comando interno a kernel.sys y no necesita ningún otro
  archivo para ejecutarse.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    break=on   Los programas se puede anular mediante CTRL+C (STRG+C)
    break=off  Los programas no se pueden anular mediante CTRL+C (STRG+C)
