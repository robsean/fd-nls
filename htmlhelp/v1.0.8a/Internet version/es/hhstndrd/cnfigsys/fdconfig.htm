Comando: fdconfig.sys

  FreeDOS lee el archivo CONFIG.SYS / FDCONFIG.SYS al inicio para
  configurar el sistema DOS. Se lee antes de que se ejecute el archivo
  AUTOEXEC.BAT.
  FreeDOS primero intenta usar FDCONFIG.SYS (e ignora CONFIG.SYS). Si 
  FDCONFIG.SYS no existe, FreeDOS utilizará CONFIG.SYS en su lugar.
  CONFIG.SYS / FDCONFIG.SYS son archivos de texto. Se pueden crear o editar
  con un editor de texto como EDIT o EDLIN.
  FDCONFIG.SYS / CONFIG.SYS siempre debe de estar en el directorio raíz del
  medio de arranque.

Sintaxis:

  - ninguna -

Opciones:

  - ninguna -

Comentarios:

  Al iniciar DOS, encontrará que solo admite un subconjunto de dispositivos
  disponibles en muchas computadoras. Para admitir dispositivos adicionales
  y funciones avanzadas, se debe cargar el controlador específico del
  dispositivo y el software residente en memoria. Esto permite que el 
  núcleo se extienda fácilmente para admitir hardware no disponible
  actualmente y aprovechar mejor el hardware instalado sin desperdiciar
  recursos en computadoras que no lo tienen.
  Este software generalmente se carga durante la fase de inicialización del
  núcleo, con detalles que describen que cargar expresados en el archivo
  CONFIG.SYS. El núcleo del FreeDOS primero buscará un archivo llamado
  FDCONFIG.SYS, si existen, se usará en lugar de CONFIG.SYS; Esto permite
  que el núcleo de FreeDOS coexista y se configure de manera diferente a
  otro núcleo de DOS. Hay opciones adicionales disponibles para ajuntar
  otros aspectos del comportamiento del núcleo.
  Hay un conjunto especial de comandos para usar en el archivo CONFIG.SYS /
  FDCONFIG.SYS, vea los enlaces a continuación.
  Los comandos de solicitud de DOS y ARCHIVOS POR LOTES generalmente no se
  pueden usar en CONFIG.SYS / FDCONFIG.SYS. Vea también el comando MENU.
  Si presiona "F5" mientras arranca el sistema, se ignoran los comandos
  CONFIG.SYS / FDCONFIG.SYS y AUTOEXEC.BAT.
  Esto es útil si tiene un controlador instalado que se bloquea al arrancar
  el sistema.
  Si presiona "F8" mientras arranca el sistema, cada comando en CONFIG.SYS/
  FDCONFIG.SYS y en AUTOEXEC.BAT debe confirmarse con "si" o "no". Esto es
  útil si desea averiguar qué controlador falla.

Ejemplos:

  - ninguno - busque los comandos individuales.
