Comando: fileshigh

  FILESHIGH le dice a FreeDOS cuántos archivos debería poder abrir de forma
  simultánea.
  FILESHIGH es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  fileshigh=n
    Donde n es el número de archivos, en el rango de 8-255. La
    configuración predeterminada es 8, pero un buen número para la mayoría
    de los sistemas modernos es 20-40. Algunos programas pueden necesitar
    una configuración más alta para funcionar correctamente; dichos
    programas generalmente le darán el valor que se necesita en su
    documentación.

Opciones:

  - ninguna -

Comentarios:

  FILESHIGH especifica cuántos archivos se pueden abrir a la vez (reserva
  la memoria necesaria para admitir la apertura de tantos archivos).
  Nota: existen otras restricciones, por lo que es posible que un programa
  determinado no pueda abrir esta cantidad. Un buen número es 20, aunque
  algunos programas sugieren / requieren 30, 40 o incluso 255.
  FILESHIGH es como FILES, excepto que primero intenta cargarse en la
  memoria de alto nivel (si no se carga en la memoria convencional).
  Nota: El pedido de carga de dispositivos puede tener un gran impacto en
  la cantidad de memoria libre disponible. En general, intente cargar
  programas grandes (en uso de memoria) en la memoria de alto nivel como
  primera opción.
  Importante: Debe tener un administrador de memoria de alto nivel como
  (HIMEM) o HIMEMX o JEMMEX instalado (antes device=himem.exe or
  device=himemx.exe en CONFIG.SYS / FDCONFIG.SYS) antes de utilizar esta
  opción.
  FILESHIGH es un comando interno de kernel.sys y no necesita ningún otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    fileshigh=20
