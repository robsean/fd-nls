Comando: shellhigh

  SHELLHIGH le dice al shell de FreeDOS que comando ejecutar.
  SHELLHIGH es un comando de CONFIG.SYS / FDCONFIG.SYS.

Sintaxis:

  shellhigh=[unidad][ruta]nombre_archivo [opciones]
            unidad          La letra de unidad, por ejemplo C:
            ruta            El directorio, por ejemplo \ejemplo\
            nombre_archivo  El shell de comandos para ejecutar. Por
                            ejemplo, C:\ejemplo\COMMAND.COM

Opciones:

  opciones  Estas se pasan al shell de comandos.

Comentarios:

  SHELLHIGH indica al shell como usarse; a menudo se usa para alterar el
  comportamiento de COMMAND.COM. Nota: es command.com que procesa
  AUTOEXEC.BAT; Al usar la opción SHELL, puede hacer que command.com
  procese un archivo con nombre diferente (como FDAUTO.BAT por coexistir
  con otro DOS usando diferentes opciones de configuración) o ejecutar un
  intérprete de comandos completamente diferente como 4DOS o una variante
  del sh (unix). SHELLHIGH es como SHELL, excepto que intenta cargar
  primero el comando SHELL (COMMAND.COM) en memoria de alto nivel (en caso
  de que no deba cargarlo en la memoria convencional).
  Nota: El pedido de carga de dispositivos puede tener un gran impacto en
  la cantidad de memoria libre disponible. En general, intente cargar
  programas grandes (en uso de memoria) en la memoria de alto nivel
  primero.
  Importante: Debe tener un administrador de memoria de alto nivel como
  (HIMEM), HIMEMX o JEMMEX instalado (device=himem.exe o device=himemx.exe
  en CONFIG.SYS / FDCONFIG.SYS) antes de usar esta opción.
  SHELLHIGH es un comando interno de kernel.sys y no necesita ningún otro
  archivo para funcionar.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    shellhigh=C:\4DOS.COM /E:256 /P
    shellhigh=C:\FDOS\bin\command.com C:\FDOS\bin /E:1024 /P=C:\fdauto.bat
