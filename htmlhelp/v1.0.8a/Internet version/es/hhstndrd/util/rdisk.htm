Comando: rdisk

  RDISK es un controlador del disco RAM en DOS como tdsk (Turbodisk).

Sintaxis:

  rdisk [/Sn] [/:L]

Opciones:

  /Sn  Especifica un tamaño designado a disco RAM en megabytes de memory
       XMS. Se cojeran valores del 2 al 2047. /S1024 o mas crea un 1- a 2-
       GIGABYTE de disco de RAM! Si /S se omite o invalida, un disco de 
       RAM de 25 MB es creado por defecto. Para el antiguo controlador
       (ROM-DOS etc.) sólo se puede usar /S2 hasta /S60.
  /:L  Especificar la letra de la unidad de DOS designada para los archivos
       de acceso RDISK. L puede estar disponible para alguna letra de 
       unidad desde A a Z, ejemplo /:N asigna a la unidad N: todos los
       archivos RDISK. Si la letra de la unidad es damasiado alta o está
       en uso ahora , RDISK será obortado, y los usuarios pueden necesitar
       "LASTDRIVE=" en CONFIG.SYS para seleccionar más unidades.
       Si RDISK es leido por CONFIG.SYS, o si /: está omitido, la
       siguiente letra de unidad libre será usada.

Comentarios:

  RDISK crea una unidad de disco "rápida" usando 2 Megabytes a 2 GIGABYTES
  de la memoria XMS! Leido como un controlador del sistema en CONFIG.SYS,
  o puede ser leido después en AUTOEXEC.BAT o como momando del usuario.
  DOS puede copiar los programas criticos, dados de archivos, y otros 
  directorios que necesitan acceso rápido (compila los archivos TEMP, base
  de datos, etc.) al disco de RAM, donde ustedes pueden leer y escribir
  en la ¡memoria rápida! si es leido después de CONFIG.SYS,
  el archivo RDISK puede asignar alguna letra unidad libre de DOS para ser
  usada como el lanzador "/:". RDISK funciona con los controladores XMS
  V2.0 o V3.0 (60-MB máximo con V2.0 XMS). Usando sólo 656 a 752 bytes de
  memoria superior dependiendo de la configuración del sistema, y también
  se puede cargar en 640K de memoria de DOS. RDISK es una pequeña unidad
  "cargalo y ovidase de el" para su uso cuando no se necesita cambiar el
  tamaño u otras características. Para mas información lea "readme.txt" en
  drivers.zip.

Ejemplos:

  EN CONFIG.SYS / FDCONFIG.SYS:
    Un ejemplo del archivo CONFIG.SYS usando V3.70+ UMBPCI y XMGR 
    es como lo siguiente:
      SHELL=C:\DOS\COMMAND.COM C:\DOS /E:512 /P
      DEVICE=C:\BIN\UMBPCI.SYS
      DEVICE=C:\BIN\XMGR.SYS /W
      DOS=HIGH,UMB
      DEVICE=C:\BIN\JEMM386.EXE I=B000-B7FF X=C800-EFFF NOEMS   ;Opcional
      DEVICEHIGH=C:\BIN\UIDE.SYS /S500 /D:CDROM1                ;O UIDEJR
      DEVICEHIGH=C:\BIN\RDISK.COM /S250                         ;Opcional
          ..
          ..  Etc.
          ..
  Un ejemplo de AUTOEXEC.BAT:
      LH C:\BIN\RDISK.COM /S500 /:Q     [Leer RDISK, asigna unidad Q:]
         ...
         ...
         ...
      MD Q:\DOS                         [Crear directorios de unidad RAM]
      MD Q:\USERPROG
      MD Q:\ ...
      COPY C:DOS\*.* Q:\DOS             [copia archivos DOS a unidad RAM]
      COPY C:\USERPROG\*.* Q:\USERPROG  [copia programas a unidad RAM]
      COPY C:\...\*.* Q:\...            [copia otros archivos, necesitados]
         ...
         ...                            [Opcional: verifica archivos RAM]
         ...
      PATH Q:\DOS;Q:\USERPROG;Q:\...    [dile a DOS "¡nuevos directorios!"]
